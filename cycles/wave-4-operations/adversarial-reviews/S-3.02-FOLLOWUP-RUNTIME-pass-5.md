## Novel Pass-5 Findings

### F-LP5-LOW-1 — extract_source_names_recursive / extract_source_names_shallow do NOT walk PipeStage::Join sources for Ast::Pipe

**Severity:** LOW
**Standing Rule 3 sub-rule:** #3 sub-clause (b) violation — sibling-not-updated
drift (C-LOCAL-001 prior fix not propagated to materialization walkers)
**Confidence:** HIGH

**Locations:**
- `materialization.rs:827-829` — `extract_source_names_shallow`, used for
  fan-out target resolution
- `materialization.rs:874-876` — `extract_source_names_recursive`, used for
  security gate via `check_internal_table_capabilities`

**Sibling correct code:** `explain.rs:489-499` — has the C-LOCAL-001 fix
walking `PipeStage::Join` sources.

**Exploit path:** Pipe-mode parser admits internal tables as JOIN sources
(`pipe_parser.rs:414-432`). A query like:

```
armis_devices | join prism_audit on host == id
```

parses successfully and produces `PipeStage::Join` with
`SourceRef.raw = "prism_audit"`. Neither `extract_source_names_shallow` nor
`extract_source_names_recursive` walks `PipeStage::Join` sources, so
`prism_audit` is not extracted. The security capability gate at
`check_internal_table_capabilities` never sees it.

**Why LOW (not MEDIUM/HIGH):**

1. Currently unexploitable — pipe-mode materialization at
   materialization.rs:556-564 returns union of materialized batches, NOT a
   DataFusion JOIN. Layer 1 misses `prism_audit` but no scan occurs (fan-out
   only sees `armis_devices`), so no Layer 2 trigger results.
2. Latent gap — converts to a real capability-gate bypass once pipe-mode JOIN
   is wired through DataFusion in S-3.06+.
3. Sibling-not-updated — same structural pattern as F-LP3-CRIT-1 / F-LP4-MED-1
   lineage, now applied to pipe mode.

**Recommendation:** Mirror the C-LOCAL-001 fix into both materialization.rs
walker functions. 3-line addition each:

```rust
for stage in &pipe.stages {
    if let PipeStage::Join(js) = stage {
        names.insert(js.source.raw.clone()); // extract_source_names_shallow
        // OR: names.push(js.source.raw.clone()); // extract_source_names_recursive
        // Add C-LOCAL-001 anchor comment mirrored from explain.rs:489-499
    }
}
```

Add 1 mirror test exercising `Ast::Pipe` with `PipeStage::Join` referencing
`prism_audit`.

**Lineage trace:** Standing Rule 3 #3 sub-clause (b); S-7.01 same-pattern
recurrence; sibling lineage F-LP3-CRIT-1 → F-LP4-MED-1 → F-LP5-LOW-1.
