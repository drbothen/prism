### Stale Documentation

Two documents contain information that does not match the current codebase:

1. **`docs/PROFILING_FINDINGS.md` Finding #6:** Claims Ping "only checks that the struct field is non-nil" and "does not make an API call." This is WRONG for the current code. `Ping()` at api.go:307-328 issues `QueryV2` with limit=1. The document describes an earlier version of the code.

2. **`internal/crowdstrike/README.md`:** References `NewClient(clientID, clientSecret, region string)` and `config.SourceConfig` -- neither of these exist in the code. The actual constructor is `NewHTTPClient(cfg Config)` and the config type is `config.CrowdStrikeConfig`. The README describes a planned or earlier API surface. The code example `source.client.FetchAlerts` would not compile (client is unexported).

3. **`docs/PROFILING_FINDINGS.md` Finding #5:** Claims "only 2 of 10 packages have tests." Actual count: 3 of 9 non-trivial packages have tests (crowdstrike, health, profiling). The document was written before crowdstrike tests existed or uses a different package counting scheme.

4. **`docs/PROFILING_FINDINGS.md` Finding #7:** Claims "double JSON serialisation." Pass 3 R2 verified this is technically accurate in terms of CPU work (two `json.Marshal` calls) but the `json.RawMessage` approach avoids re-encoding the inner data. The finding overstates the impact.
