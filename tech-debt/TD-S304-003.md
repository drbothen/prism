# TD-S304-003: query_context dual-recording (original + expanded query)

**Story:** S-3.04
**Status:** open
**Severity:** tech_debt

## Description

BC-2.11.009 postcondition requires that both the original query string and the
expanded query string are recorded in `query_context` for audit transparency.
This is not implemented because `query_context` is a future capability.

## Current Behavior

`AliasResolver::expand` returns the expanded query but does not record the
original query anywhere. The audit trail only captures what the engine executes,
not the original alias-containing query.

## Required Fix

When `AliasResolver::expand` is called from the query engine:
1. Store `original_query = input` in the query context.
2. Store `expanded_query = result` in the query context.
3. Emit a structured log entry recording both values (DI-004 audit).

## Tests Deferred

- `test_BC_2_11_009_query_context_records_original_and_expanded` — tests
  `expand()` returns Ok/Err but does not verify dual-recording (no
  query_context infrastructure yet).
