# TD-S305-002: DEC-020 cross-client fetch ordering requires fetch scheduler

**Filed:** 2026-05-06
**Story:** S-3.05
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`test_BC_2_07_002_dec020_cross_client_fetch_ordering_alphabetical` verifies that when
the cursor cap is reached during cross-client fan-out, clients are processed in alphabetical
order by `client_id`.

## Why Deferred

The alphabetical ordering is a fetch scheduler policy — it lives in the fan-out orchestration
layer (S-5.01 scope), not in `cursor.rs`. The `QueryCursorRegistry` stores per-cursor
result sets but has no awareness of fan-out scheduling order.

## Resolution Path

During S-5.01 (MCP tool integration): implement fetch scheduler with alphabetical client
ordering and cover this invariant with an integration test.

`crates/prism-query/src/tests/pagination_tests.rs::test_BC_2_07_002_dec020_cross_client_fetch_ordering_alphabetical`
