# TD-S302-004: prism_audit table enumeration via schema introspection (design decision)

**Filed:** 2026-05-06
**Story:** S-3.02 (PR #129 review, SEC-P129-001)
**Severity:** tech_debt
**Status:** open / deferred — requires PO design decision

## Finding

The `prism_audit` internal table (registered in `internal_tables.rs`) is enumerable
via the DataFusion catalog API — an MCP caller with `query` tool access can run
`SHOW TABLES` or query the information schema to discover that `prism_audit` exists
and infer its schema. This is an information disclosure concern (CWE-200): the audit
table name and structure should not be discoverable via the same tool interface that
exposes user-facing sensor data.

## Location

- `crates/prism-query/src/internal_tables.rs` — `RocksDbTableProvider` registration
- `crates/prism-query/src/engine.rs` — table registration sequence
- BC-2.11.005 — Ephemeral Materialization (table registration step)

## Design Options

1. **Separate catalog namespace:** Register internal tables in a private DataFusion
   catalog that is not accessible to the MCP query interface.
2. **Schema filter:** Filter `SHOW TABLES` / information_schema results to exclude
   tables prefixed with `prism_`.
3. **Separate DataFusion context:** Use one `SessionContext` for user queries and a
   separate one for internal audit writes. No cross-context table visibility.
4. **Accept as-by-design:** Document that `prism_audit` table existence is not a secret
   (audit table names are common knowledge). Restrict row-level access instead.

## Why Deferred

Requires PO/architecture decision on the right approach. Option 4 may be acceptable
given that the audit table is read-protected by RBAC at the query execution layer.
Options 1–3 require DataFusion catalog architecture changes that affect multiple stories.

## Required Action

PO to decide on approach and update BC-2.11.005 or create a new BC for internal table
access control. File as a BC amendment request if Option 1, 2, or 3 is chosen.

**Target story:** Security hardening pass or dedicated S-3.X story.
