# TD-S304-VP013-001: Populate alias stores in VP-013 proptest harnesses

**Story:** S-3.04
**Status:** open
**Severity:** tech_debt
**Filing:** pass-1 review CR-019

## Description

The VP-013 concrete tests and proptest harnesses test cycle detection against empty stores.
An empty store cannot exercise the full DFS traversal because there are no cross-references
to follow — the traversal terminates immediately.

## Current Behavior

Tests call `AliasResolver::detect_cycle("A", definition, &empty_store)` which can only
detect self-references (`@A` in `A`'s body). It cannot detect multi-hop cycles
(A → B → A) when B is not in the store.

## Required Fix

In VP-013 proptest harnesses:
1. Build a store with 2–5 aliases using proptest-generated names and cross-references.
2. Call `detect_cycle` with the last alias pointing back to the first.
3. Assert the cycle is detected.

In concrete tests:
- Add a test where B is in the store and B's body references A (mutual cycle A→B→A).
