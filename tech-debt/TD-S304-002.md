# TD-S304-002: alias.write capability gate compile-time feature flag integration

**Story:** S-3.04
**Status:** open
**Severity:** tech_debt

## Description

`create_alias` does not check the `alias.write` capability via `FeatureFlagEvaluator`.
The `alias_capability::check_alias_write` function exists but is not called from
`create_alias` or `create_alias_with_clients` because `FeatureFlagEvaluator` is not
available in the tool handler context.

## Current Behavior

`create_alias` and `delete_alias` skip the capability gate. Any MCP caller can create
or delete aliases regardless of `alias.write` runtime configuration.

## Required Fix

1. Thread `&FeatureFlagEvaluator` and `CompileTimeGate` into `create_alias_with_clients`
   and `delete_alias`.
2. Call `alias_capability::check_alias_write(scope, evaluator, compile_gate)` before
   any mutation.
3. Add `#[cfg(feature = "alias-write")]` compile-time gate in the query crate.

## Tests Deferred

- `test_ac12_write_capability_gate` — currently tests `check_alias_write` in isolation
  rather than through the full tool handler flow.
