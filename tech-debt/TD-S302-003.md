# TD-S302-003: Constant-array allocation in virtual_fields (post-stub perf)

**Filed:** 2026-05-06
**Story:** S-3.02 (PR #129 review, F-PR129-CR-016)
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`virtual_fields.rs` allocates the reserved-field name list as a `Vec` or similar
heap-allocated structure per call (or per call path through `remove_spoofed_virtual_columns`).
These names are compile-time constants and could be stored as `&'static [&'static str]`
to avoid heap allocation in the hot path.

## Location

- `crates/prism-query/src/virtual_fields.rs` — `remove_spoofed_virtual_columns` and
  related helpers that iterate over the reserved field name set.

## Why Deferred

The current implementation is a stub (S-3.02 scope). Per-call allocation cost is
negligible during stub phase. Performance optimization belongs in a later story after
the full pipeline is wired and the hot path is identified through profiling.

## Fix Scope

1. Define a `const RESERVED_VIRTUAL_FIELDS: &[&str]` containing all reserved names.
2. Replace per-call `vec![...]` construction with a slice reference.
3. Verify all virtual_fields tests still pass.

**Target story:** S-3.X pipeline wiring or dedicated performance pass.
