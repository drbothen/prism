# TD-S302-002: parse_bytes_from_error_msg complexity simplification

**Filed:** 2026-05-06
**Story:** S-3.02 (PR #129 review, F-PR129-CR-014)
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`parse_bytes_from_error_msg` in `crates/prism-query/src/memory.rs` uses a
reverse-iteration approach to extract a byte count from a DataFusion error message.
While correct, the implementation is more complex than necessary. A simpler approach
using a regex or split-based parse would be clearer and easier to maintain.

## Current Implementation

```rust
fn parse_bytes_from_error_msg(msg: &str) -> Option<u64> {
    let bytes_pos = msg.find(" bytes")?;
    let before = &msg[..bytes_pos];
    let digits: String = before.chars().rev().take_while(|c| c.is_ascii_digit()).collect();
    if digits.is_empty() { return None; }
    let reversed: String = digits.chars().rev().collect();
    reversed.parse::<u64>().ok()
}
```

## Simpler Alternative

```rust
fn parse_bytes_from_error_msg(msg: &str) -> Option<u64> {
    // DataFusion format: "... Failed to allocate NNN bytes for ..."
    let before_bytes = msg.split(" bytes").next()?;
    before_bytes.split_whitespace().last()?.parse::<u64>().ok()
}
```

## Why Deferred

The current implementation is correct and well-tested (3 tests cover it). Simplification
is a readability improvement only. No correctness risk warrants blocking S-3.02 delivery.

## Fix Scope

1. Rewrite `parse_bytes_from_error_msg` using `split_whitespace().last()` approach.
2. Verify all 3 existing unit tests still pass.

**Target story:** Maintenance sweep.
