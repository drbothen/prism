# TD-S305-001: AC-3 cursor expiry test requires clock injection

**Filed:** 2026-05-06
**Updated:** 2026-05-07 (OBS-009 formalization — error code corrected to E-QUERY-012; coverage gap scope clarified)
**Story:** S-3.05
**Severity:** P2 / test-coverage-gap
**Status:** open / deferred

## Finding

AC-3 (cursor 60s expiry) is currently NOT verified by any active unit or integration test.
The unit test for cursor TTL expiry in `crates/prism-query/src/tests/pagination_tests.rs`
is marked `#[ignore]` pending clock injection infrastructure. No integration tests for S-3.05
exist yet. As a result, the AC-3 postcondition — "Given a cursor token that is 61 seconds old,
When `next_page(token)` is called, Then `E-QUERY-012` (`PrismError::CursorExpired`) is returned
and the cursor entry is removed from the registry" — has zero active test coverage.

**Error code note:** fix-pass-16 (commit d36ecf22) renumbered `CursorExpired` to E-QUERY-012
(was incorrectly E-QUERY-006 per fix-pass-15; incorrectly E-QUERY-004 in the original v1.10
story text). AC-3 in S-3.05 v1.11 now correctly cites E-QUERY-012. The test expectation should
assert `PrismError::CursorExpired` variant (not a raw error code string).

## Why Deferred

The test creates a cursor and immediately calls `next_page()` — the cursor is ~0ms old,
not 61 seconds. Making the test pass requires either:
1. A `ClockSource` trait (`fn now() -> Instant`) injected into `CursorRegistry` as a generic
   parameter or `Arc<dyn ClockSource>`, with `RealClock` for production and `MockClock` for tests, OR
2. `tokio::time::pause()` + `tokio::time::advance()` in an async test context.

## Coverage Gap Scope

- `crates/prism-query/src/tests/pagination_tests.rs` — the `#[ignore]`'d expiry test
- `cursor.rs` — `CursorRegistry::next_page` expiry path (line ~244 per TD-VSDD-071)
- Background eviction task (30s wake interval) — also untestable without clock injection
- Story-level traceability matrix should reflect this gap until clock injection is implemented

## Resolution Path

Introduce a `ClockSource` trait into `cursor.rs`. Inject a `MockClock` in tests. Once injected:
- Remove the `#[ignore]` attribute
- Assert `CursorRegistry::next_page(expired_token)` returns `Err(PrismError::CursorExpired)` (E-QUERY-012)
- Assert cursor is removed from registry after expiry call
- Assert background eviction task removes entries after advancing mock clock past 60s TTL

## Test Location

`crates/prism-query/src/tests/pagination_tests.rs` — cursor TTL expiry test (currently `#[ignore]`)
