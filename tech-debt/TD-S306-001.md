# TD-S306-001: sensor_verbs always empty in WriteVerbRegistry

**Filed:** 2026-05-06
**Story:** S-3.06 (PR #130 review, F-PR130-CR-002)
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`WriteVerbRegistry::sensor_verbs()` always returns an empty `Vec` because
`WriteVerbSource` has no `all_sensors() -> Vec<String>` method. When
`parse_pipe_with_write` needs to populate the E-QUERY-023 suggestion list
for an unknown terminal verb, it falls back to `registry.all_verbs()` when
`sensor_verbs` is empty ‚Äî which is functional (suggestions list all verbs)
but less precise than per-sensor suggestions.

## Location

- `crates/prism-query/src/write_verb_registry.rs` ‚Äî `WriteVerbRegistry::verbs_for_sensor`
- `crates/prism-query/src/pipe_parser.rs` ‚Äî `parse_pipe_with_write` E-QUERY-023 suggestion path

## Why Deferred

Fixing this requires extending `WriteVerbSource` with an `all_sensors() -> Vec<String>`
method. That is a larger interface change affecting all implementors of the trait,
including the live `WriteEndpointRegistry` in S-3.07 scope.

The workaround (suggestions list all verbs regardless of sensor) is functional and
correct ‚Äî it never hides a valid verb from the user. It just doesn't narrow suggestions
to the specific sensor's verbs. This is not security-blocking.

## Fix Scope

1. Add `all_sensors() -> Vec<String>` to `WriteVerbSource` trait.
2. Implement in `WriteVerbRegistry::from_source` to populate a sensor‚Üíverbs map.
3. Update `verbs_for_sensor` to return the correct subset.
4. Update `parse_pipe_with_write` E-QUERY-023 suggestion path to use the per-sensor list.

**Target story:** S-3.07 or a dedicated maintenance story.
