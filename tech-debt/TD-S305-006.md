# TD-S305-006: TTL absolute measurement test requires clock injection

**Filed:** 2026-05-06
**Story:** S-3.05
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`test_BC_2_07_003_ttl_measured_from_created_at_not_from_last_access` verifies that cache
TTL is measured from `created_at` (absolute), not reset on each access. The test body has
a `todo!()` at line 344.

## Why Deferred

The test constructs a `CacheEntry` directly and calls `is_expired()` on a freshly-created
entry (should return false). The assertion that TTL is NOT reset on access requires:
1. Inserting an entry
2. Advancing time past its TTL
3. Confirming `get()` returns None even after multiple prior hits

Step 2 requires either `std::thread::sleep` (slow/flaky) or `tokio::time::pause()`.

## Implementation Note

`CacheEntry::is_expired()` is correctly implemented as `created_at.elapsed() > ttl` —
it does NOT use last-access time. The invariant is semantically correct; only the test
automation (clock advancing) is deferred.

## Resolution Path

During cache integration pass: use `tokio::time::pause()` + `advance()` or inject a
clock abstraction to verify the absolute TTL invariant.
