# TD-S305-005: Cursor cleanup no-leak test requires clock injection

**Filed:** 2026-05-06
**Story:** S-3.05
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`test_BC_2_07_002_expired_cursor_removed_from_registry_no_leak` verifies that calling
`next_page()` on an expired cursor removes it from the registry (no memory leak), and
that `active_count()` returns 0 after expiry.

## Why Deferred

The test body has a `todo!()` marker. After `create()` returns a cursor token, the cursor
is ~0ms old — not expired. The test cannot advance time without a clock abstraction.

## Resolution Path

Same path as TD-S305-001: inject a clock abstraction into `QueryCursorRegistry::new()` or
use `tokio::time::pause()` in an async test context. Both `evict_expired()` and `next_page()`
would respect the injected clock for expiry checks.
