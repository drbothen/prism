# TD-S305-004: EC-07-022 server-side cursor expiry requires sensor adapter

**Filed:** 2026-05-06
**Story:** S-3.05
**Severity:** tech_debt
**Status:** open / deferred

## Finding

`test_BC_2_07_002_ec07022_server_side_cursor_expiry_partial_results` verifies that when
a sensor API cursor expires server-side during a long multi-page fetch, partial results from
pages already retrieved are materialized and the error is reported in `sensor_errors`.

## Why Deferred

The test body has a `todo!()` marker. Server-side cursor expiry is a sensor adapter concern:
- The `QueryCursorRegistry` holds client-side pagination state, not sensor API state.
- The sensor adapter must catch the server-side expiry (HTTP 4xx/expired token response)
  and propagate it as `PrismError::Sensor` with partial results.
- This requires DTU clone implementation (S-DTU scope).

## Resolution Path

During DTU sensor adapter implementation: cover EC-07-022 with an integration test
against a DTU clone that simulates server-side cursor expiry mid-fetch.
