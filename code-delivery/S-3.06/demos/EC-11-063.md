# EC-11-063 — Unrecognised Verb in Terminal Pipe Position

**Story:** S-3.06 v1.7 | **BC:** BC-2.11.004 | **Status:** PASS

## Edge Case

Unrecognised verb in terminal pipe position returns `E-QUERY-023` with available-verb
suggestion list for the source sensor.

## Test Names

- Integration: `test_ac5_unknown_verb_suggestion`
- Unit (pub-crate): `test_BC_2_11_004_parse_pipe_with_write_unknown_verb`
- Unit (pub-crate): `test_BC_2_11_004_parse_pipe_with_write_empty_registry_any_verb_023`
- Error constructor: `test_BC_2_11_004_error_023_message_contains_code_verb_and_suggestions`

## Evidence

### Unit — unknown verb with populated registry

```
test test_BC_2_11_004_parse_pipe_with_write_unknown_verb ... ok
```

Query: `FROM crowdstrike_hosts | where x = 1 | nonexistent_verb`
Registry: `["contain", "tag"]`

Assertions:
- `result.is_err()`
- `result.unwrap_err()[0].message.contains("E-QUERY-023")`

### Unit — empty registry (EC-11-065 overlap)

```
test test_BC_2_11_004_parse_pipe_with_write_empty_registry_any_verb_023 ... ok
```

With an empty registry, ANY terminal identifier becomes E-QUERY-023.

### Error constructor test

```
test test_BC_2_11_004_error_023_message_contains_code_verb_and_suggestions ... ok
```

```rust
let err = ParseError::unknown_write_verb(0, "nonexistent", &["contain", "tag"]);
assert!(err.message.contains("E-QUERY-023"));
assert!(err.message.contains("nonexistent"));
```

### Error message format

With available verbs:
```
E-QUERY-023: unknown write verb 'nonexistent_verb' — available verbs: contain, tag
```

Without available verbs (empty registry):
```
E-QUERY-023: unknown write verb 'VERB' — no write verbs are registered for this sensor
```

## Result

PASS — E-QUERY-023 returned with suggestion list; verbs from the sensor's registry
included in error message.
