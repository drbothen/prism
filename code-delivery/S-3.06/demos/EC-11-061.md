### Guard function

```rust
pub(crate) fn is_internal_prism_table(table_name: &str) -> bool {
    table_name.starts_with("prism_")
}
```

Guard fires before `check_unbounded_write` — if `E-QUERY-010` fires, `E-QUERY-022`
is not checked (the DmlNode is never produced).

### Architecture compliance

Per story: "E-QUERY-010 MUST be emitted at parse time for SQL DML targeting prism_*
tables — do not defer this check to execution time." Confirmed: the check occurs in
`sql_parser.rs`, not in any execution-layer code.
