### Oversized query (security guard)

```
test test_BC_2_11_004_oversized_write_query_rejected_before_parse ... ok
```

A DELETE query with a 66,000-character table name is rejected before reaching the
parser (query size limit 64KB enforced by `security.rs`). No panic, no unbounded
allocation.

### Fuzz target (CI)

The cargo-fuzz target `vp021_parse_fuzz` in `fuzz/fuzz_targets/vp021_parse_fuzz.rs`
is the formal fuzz vehicle. The corpus above represents concrete seeds added to cover
write-mode inputs. The fuzz target is exercised in CI nightly runs per the existing
`just fuzz-local prism-query vp021_parse_fuzz` workflow.
