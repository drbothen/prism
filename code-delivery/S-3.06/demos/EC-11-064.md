# EC-11-064 — Write Verb in Filter Mode

**Story:** S-3.06 v1.7 | **BC:** BC-2.11.004 | **Status:** PASS

## Edge Case

Write verb used in filter mode (expression begins with predicate, no FROM keyword)
produces a hard parse-time rejection; error indicates writes not permitted in filter mode.

### Key behavior

The integration tests via public API (`PrismQlParser::parse`) do not assert the
specific error code because the public API does not inject a registry. The unit test
`test_BC_2_11_004_reject_write_verbs_in_filter_with_verb_in_input` directly calls
`reject_write_verbs_in_filter("severity_id >= 4 | contain", &registry)` with a
registry containing `["contain"]` and asserts `result.is_err()`.

### Grammar-level rejection

The rejection is NOT a post-parse semantic check. `reject_write_verbs_in_filter` scans
the input for registered write verb tokens before Chumsky parsing begins. If any
registered verb is found after a `|` in filter mode, parsing is aborted immediately.

### False-positive guard

`test_BC_2_11_004_filter_mode_field_named_contain_is_not_rejected` verifies that
`contain = 1` (field named `contain` in a predicate, no pipe) does NOT trigger the
write-mode rejection.

## Result

PASS — Filter mode hard-rejects write verbs at grammar level. Field names that happen
to match verb names are not falsely rejected.
