## Notes on 3 Passing Tests

Three tests in `bc_2_09_006_test.rs` pass without implementation:

1. `test_BC_2_09_006_data_trust_level_section_contains_canonical_text`
2. `test_BC_2_09_006_non_sensor_tool_omits_data_trust_level_and_security_note`
3. `test_BC_2_09_006_security_note_mentions_all_adversarial_field_types`

These tests exercise only `const` values and a local test helper function `build_full_sensor_desc()` that assembles a string inline. They call no `unimplemented!()` stubs. They validate that the stub API surface declares the correct constants per the BC spec. This is intentional — constants are not unimplemented behavior. They will remain passing post-implementation and serve as specification documentation.

All tests that call unimplemented stub methods panic with `unimplemented!("… — stub (Red Gate)")`.
