## Production Code Reference

**File:** `crates/prism-spec-engine/src/pipeline.rs`

The `arbitrary_json()` proptest strategy in pipeline.rs generates structurally valid
`serde_json::Value` inputs for the `proptest_extract_at_path_totality` property. This strategy
was introduced as part of the HIGH-002 closure (fix-burst-2) to ensure the totality property
covers non-trivial nested JSON shapes, not just flat scalars.

**`fan_out_batches`** is a pure chunking function. The proptest properties in
`proptest_AC_3.rs` use `proptest`'s integer strategies to generate arbitrary input lengths (1
to 1000 elements) and batch sizes (1 to 500), asserting that:
1. The total element count in all output batches equals the input count (no loss, no duplication).
2. No single output batch exceeds `batch_size` elements.

**`Interpolator::extract_references`** is tested via a round-trip property: for any template
string generated by the strategy, every `${...}` reference token returned by `extract_references`
is used to build a synthetic variable map, and a subsequent `interpolate` call with that map
must not return an error for a missing variable.
