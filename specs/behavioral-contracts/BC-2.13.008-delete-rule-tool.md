# BC-2.13.008: `delete_rule` MCP Tool — Remove Rule (Confirmation for Global Rules)

## Description

The `delete_rule` tool removes a detection rule at a specified scope, with confirmation requirements varying by scope: global rules require a two-step confirmation token (irreversible, affects all clients); client rules use standard write gating; analyst rules are deleted immediately (session-scoped). Deletion purges correlation windows and sequence trackers for the deleted rule; historical alerts generated by the rule are preserved. Deleting a client override re-exposes the global rule (if any) for that client. Gated by `detection.write` (global also requires `detection.write.global`).

## Preconditions
- The `delete_rule` MCP tool is invoked with required parameters: `rule_id` and `scope` (`global`, `client`, `analyst`)
- If `scope` is `client`: `client_id` parameter is required
- The `detection.write` capability is allowed (for `global` scope, `detection.write.global` is required)
- A rule with the given `rule_id` exists at the specified scope

## Postconditions
- For `global` scope: confirmation token required (irreversible write, BC-2.04.009); preview shows rule details and list of clients currently using this rule
- For `client` scope: standard write gating; rule removed from that client's overrides
- For `analyst` scope: immediate deletion (session-scoped, no confirmation)
- The rule is removed from the appropriate storage location
- Detection engines are rebuilt for affected scopes; stateful engine state (correlation windows, sequence trackers) associated with the deleted rule is purged
- Alerts previously generated by this rule are **not** deleted (historical record preserved)
- An audit entry is emitted (DI-004)

## Invariants
- DI-004: Audit completeness
- Deleting a client-scope override re-exposes the global rule (if one exists with the same ID) for that client
- Historical alerts are immutable; rule deletion does not cascade to alert history

## Error Cases
| Error | Condition | Behavior |
|-------|-----------|----------|
| `E-RULE-007` | No rule with given `rule_id` at specified scope | Structured error |
| `E-FLAG-001` | `detection.write` capability denied | Structured error (BC-2.04.015) |
| `E-FLAG-008` | Confirmation token expired or invalid (global scope) | Structured error with recovery guidance |

## Edge Cases
| ID | Description | Expected Behavior |
|----|-------------|-------------------|
| EC-13-027 | Delete global rule that has client-scope overrides | Global rule deleted; client overrides remain (they no longer override anything; they become standalone client rules) |
| EC-13-028 | Delete client-scope override when no global rule exists | Rule is simply removed for that client; no fallback rule takes effect |
| EC-13-029 | Delete rule while correlation windows are accumulating | Windows for the deleted rule are purged immediately |
