# S-3.4.04 — Migrate prism-dtu-cyberint tests to prism-dtu-harness

## Narrative

As a Prism integration-test maintainer, I want the `prism-dtu-cyberint` test suite
rewritten to use `prism-dtu-harness`, so that Cyberint's 12 ACs, edge cases, and
fidelity validator benefit from the same multi-tenant isolation coverage as the other
sensor DTUs.

## Objective

Rewrite all `prism-dtu-cyberint` tests — 12 ACs, edge cases, `fidelity_validator`,
and 2 TD tests — to use `HarnessBuilder`. All 12 original ACs must pass post-migration.
Add 2-org logical and network isolation tests.

## Tasks

1. Catalog all existing test functions in `crates/prism-dtu-cyberint/tests/`.

2. Rewrite `tests/harness_tests.rs`:
   - All 12 ACs use harness-hosted clone (logical mode)
   - Edge case tests preserved
   - TD tests rewritten with harness

3. Add 2-org logical isolation test: pairwise-disjoint alert/indicator sets.

4. Add network isolation test: cross-org credential mismatch → HTTP 401.

5. Update fidelity validator `base_url` from `harness.endpoints()`.

6. Add `prism-dtu-harness` dev-dep to `prism-dtu-cyberint/Cargo.toml`.

**AC-001:** All 12 original `prism-dtu-cyberint` ACs pass against the harness-hosted clone in `IsolationMode::Logical`.
