# S-3.04-FOLLOWUP-MCP-001: Wire alias tools into prism-mcp (MCP integration)

**Story:** S-3.04-FOLLOWUP-MCP-001
**Status:** open
**Parent:** S-3.04
**Priority:** P2
**Filing:** S-3.04 local adversary pass-1 F-HIGH-001
**Version:** v1.0

## Background

S-3.04 implemented the alias system (BC-2.11.008/009/013/014/015) inside `prism-query`.
The four alias tools (`create_alias`, `list_aliases`, `delete_alias`, `explain_alias`)
are implemented and tested within `prism-query` but are NOT yet registered in the MCP
`tools/list` response or dispatched by the MCP handler in `prism-mcp`.

## Required Work

1. **Register alias tools in `prism-mcp` tools/list**
   - Add `create_alias`, `list_aliases`, `delete_alias`, `explain_alias` to the
     tool manifest (name, description, inputSchema).
   - Gate `create_alias` and `delete_alias` behind the `alias.write` compile-time
     feature AND the runtime capability flag (hidden-tools pattern per BC-2.04.004).

2. **Wire MCP dispatch**
   - Add match arms for each tool name in the MCP request dispatcher.
   - Deserialise `params` → `CreateAliasInput` / `ListAliasesInput` / etc.
   - Call `create_alias_with_clients_gated`, `list_aliases`, `delete_alias_gated`,
     `explain_alias` from `prism_query::alias_tools`.
   - Thread `Arc<AliasStore>`, `Arc<ConfirmationTokenStore>`, `valid_client_ids`, and
     `capability_gate` from the MCP server state to each call.

3. **Lifecycle management**
   - Load `AliasStore` at server startup from `aliases.toml` (path from config).
   - Wrap in `Arc<Mutex<AliasStore>>` for concurrent MCP request handling.
   - Handle `aliases.toml` absent at startup (use `AliasStore::empty`).

4. **Tests**
   - Integration test: MCP `tools/list` includes alias tools when alias.write is
     enabled.
   - Integration test: two-step create_alias flow over MCP (token round-trip).
   - Integration test: two-step delete_alias flow over MCP.

## Architecture Mapping

- `prism-mcp` crate: `src/server.rs` (tool registration), `src/dispatch.rs` (handler)
- `prism-query` crate: `alias_tools.rs` (already implemented)
- BC-2.11.008 / BC-2.11.013 / BC-2.11.014 / BC-2.11.015
- Feature flag: `alias-write` (added in S-3.04 F-HIGH-002)

## Deferral Rationale

Implementing MCP wiring in S-3.04 would require cross-crate changes to `prism-mcp`
that are larger in scope than the remaining S-3.04 adversary fixes. Option B (formal
follow-up story) was chosen per adversary recommendation.
