# S-1.01 — prism-core: Foundational Types

## Narrative

As a Prism platform implementer, I want shared foundational types (TenantId, PrismError,
StorageDomain, ColumnOptions, ConfigSnapshot) defined in a single root crate, so that
every downstream crate can depend on validated, well-typed primitives without circular
dependencies or duplicated definitions.

## Objective

Establish the prism-core crate as the dependency root for the entire Cargo workspace.
All other crates depend on prism-core; prism-core depends on nothing internal. This
story establishes the workspace layout, the canonical error taxonomy, tenant isolation
types, and the storage domain enumeration that drives RocksDB column family naming.

## Tasks

1. Create the Cargo workspace at the repo root (`Cargo.toml` with `[workspace]` and
   `members = [...]`). Add `prism-core` as the first member.

2. Create `crates/prism-core/Cargo.toml` with dependencies: `thiserror`, `serde`,
   `regex` (for TenantId validation), `uuid` (for ID newtypes), `tracing`.

3. Implement `TenantId` newtype in `crates/prism-core/src/tenant.rs`:
   - Inner type: `Arc<str>` (cheap clone, immutable)
   - Constructor: `TenantId::new(s: &str) -> Result<Self, PrismError>`
   - Validation regex: `^[a-zA-Z0-9_-]{1,64}$` (compiled once via OnceLock)
   - Reject: empty string, length > 64, any character outside `[a-zA-Z0-9_-]`
   - Derive: `Clone, Debug, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`

4. Implement foundational newtypes in `crates/prism-core/src/types.rs`:
   - `ClientId`: newtype wrapping `TenantId` (alias for client identification in MSSP context)
   - `AnalystId`: newtype wrapping `Arc<str>`, validated same as TenantId (MSSP analyst identifier)
   - `SeverityId`: newtype wrapping `u32` (OCSF severity_id values 0-5)
   - `Timestamp`: newtype wrapping `chrono::DateTime<Utc>` (all timestamps in Prism are UTC)
   - All derive: `Clone, Debug, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`

5. Implement `PrismError` enum in `crates/prism-core/src/error.rs` using `thiserror`:
   - Cover all 90+ error codes from the error taxonomy
   - Category prefixes: `E-AUTH-*`, `E-SENSOR-*`, `E-OCSF-*`, `E-CRED-*`, `E-FLAG-*`,
     `E-STORE-*`, `E-CFG-*`, `E-MCP-*`, `E-SAFETY-*`, `E-QUERY-*`, `E-SCHED-*`,
     `E-DET-*`, `E-CASE-*`, `E-WATCH-*`, `E-SPEC-*`, `E-IOC-*`
   - Every variant's `Display` impl must emit the structured error code as the first
     token, e.g., `"E-STORE-001: RocksDB open failed: {detail}"`
   - Include variants: `InvalidTenantId`, `InvalidCredentialName`, `CursorCapExceeded`,
     `StorageDomainNotFound`, and domain-specific variants for each category

6. Implement `StorageDomain` enum in `crates/prism-core/src/storage.rs`:
   - Exactly 15 variants (matches RocksDB column families defined in S-2.01):
     `Default`, `Schedules`, `DiffResults`, `DetectionRules`, `DetectionState`,
     `Alerts`, `Cases`, `AuditBuffer`, `DirtyBits`, `Watchdog`, `Aliases`,
     `Decorators`, `InfusionCache`, `ActionState`, `PluginState`
   - Implement `StorageDomain::column_family_name(&self) -> &'static str` returning
     the snake_case string for each variant
   - Implement `StorageDomain::all() -> &'static [StorageDomain]` returning all 15
   - Derive: `Clone, Copy, Debug, PartialEq, Eq, Hash`

7. Implement `ColumnType` enum in `crates/prism-core/src/types.rs`:
   - Variants: `String`, `Integer`, `Float`, `Boolean`, `Datetime`, `Json`
   - Used by `InternalTableDescriptor` (S-2.03) for schema definitions without Arrow dependency
   - prism-query (S-3.02) converts to Arrow DataType at the query layer
   - Derive: `Clone, Copy, Debug, PartialEq, Eq, serde::Serialize, serde::Deserialize`

8. Define `CacheBackend` trait in `crates/prism-core/src/cache.rs`:
   - `async fn get(&self, domain: StorageDomain, key: &[u8]) -> Result<Option<Vec<u8>>, PrismError>`
   - `async fn set(&self, domain: StorageDomain, key: &[u8], value: &[u8]) -> Result<(), PrismError>`
   - `async fn delete(&self, domain: StorageDomain, key: &[u8]) -> Result<(), PrismError>`
   - Trait must be `Send + Sync + 'static` (same as StorageBackend)
   - Implemented by prism-storage's RocksDbBackend (S-2.01); injected into
     prism-spec-engine at startup by prism-bin. This preserves the layering invariant:
     prism-spec-engine depends on prism-core (trait), not prism-storage (implementation).
   - Used by S-1.14 (infusion Tier 3 cache) and S-1.15 (plugin KV store)

9. Implement `ColumnOptions` struct in `crates/prism-core/src/storage.rs`:
   - Fields: `ttl_seconds: Option<u64>`, `compression: bool`, `block_cache_mb: u32`
   - Provide sensible defaults via `Default` impl

10. Implement `ConfigSnapshot` shell struct in `crates/prism-core/src/config.rs`:
   - Placeholder with `version: u64` and `raw: serde_json::Value`
   - Populated by prism-spec-engine in S-1.11; this stub prevents circular deps

11. Define tracing subscriber configuration types in `crates/prism-core/src/telemetry.rs`:
   - `TracingConfig { level: tracing::Level, json_output: bool, service_name: String }`
   - `init_tracing(cfg: &TracingConfig)` function that installs the subscriber

12. Write VP-001 Kani proof in `crates/prism-core/src/proofs/tenant_id.rs`:
   - Proof 1: `TenantId::new("")` always returns `Err` (empty string rejected)
   - Proof 2: A string of 65 `'a'` characters always returns `Err` (length > 64 rejected)
   - Proof 3: A string containing `'/'` always returns `Err` (invalid character rejected)
   - Proof 4: `"acme_corp-01"` always returns `Ok` (valid input accepted)
   - Use `#[cfg(kani)]` guard; do not affect non-kani builds

13. Write unit tests in `crates/prism-core/src/tests/`:
    - Test every `StorageDomain` variant's `column_family_name()` for expected string
    - Test `StorageDomain::all()` returns exactly 15 elements
    - Test `PrismError` display for each category prefix

## Acceptance Criteria

**AC-1:** Given an empty string `""`, When `TenantId::new("")` is called, Then it returns
`Err(PrismError::InvalidTenantId)` with a message containing `"E-AUTH"`.

**AC-2:** Given a valid identifier `"acme_corp-01"`, When `TenantId::new("acme_corp-01")`
is called, Then it returns `Ok(TenantId)` and `tenant_id.as_str() == "acme_corp-01"`.

**AC-3:** Given a string with path traversal `"../etc"`, When `TenantId::new("../etc")`
is called, Then it returns `Err` (the `.` character is outside `[a-zA-Z0-9_-]`).

**AC-4:** Given the `StorageDomain` enum, When `StorageDomain::all()` is iterated, Then
exactly 15 variants are returned with distinct `column_family_name()` strings.

**AC-5:** Given any `PrismError` variant, When `format!("{err}")` is called, Then the
result begins with the structured error code (e.g., `"E-STORE-001"`), not a generic
message.

**AC-6:** VP-001 Kani proof compiles and passes with 0 verification failures when run
with `cargo kani --proof tenant_id`.

**AC-7:** Given a `TenantId` with value `"acme"`, When it is serialized to JSON and
deserialized back, Then the round-trip produces an equal `TenantId`.

**AC-8:** Given a 64-character string of valid characters, When `TenantId::new()` is
called, Then it returns `Ok` (boundary: exactly 64 is valid).

**AC-9:** Given a 65-character string of valid characters, When `TenantId::new()` is
called, Then it returns `Err` (boundary: 65 is invalid).

## Architecture Compliance Rules

- `prism-core` MUST have zero internal Prism dependencies (it is the dependency root)
- All types in this crate MUST be `Send + Sync` (required for async runtime use)
- `PrismError` MUST use `thiserror` (not `anyhow`) — callers need to match on variants
- `TenantId` MUST be cheap to clone — use `Arc<str>` not `String`
- Kani proofs MUST be gated with `#[cfg(kani)]` — they must not affect test or release builds
- The `ConfigSnapshot` struct MUST remain a shell — do not pull prism-spec-engine types in

## Notes

- The `PrismError` enum will be large (90+ variants). Group variants by category using
  `#[allow(clippy::large_enum_variant)]` if needed, but prefer boxing large payloads.
- `StorageDomain::all()` should return a `&'static [StorageDomain]` via a const array,
  not a Vec — avoids allocation in the storage initialization hot path.
- The regex for TenantId should be compiled once via `std::sync::OnceLock<Regex>` and
  reused. Do not compile per call.
- `ConfigSnapshot` intentionally has no real fields yet. S-1.11 (prism-spec-engine)
  will populate it. This stub exists only to break the dependency cycle.
- Tasks in this story MUST be executed sequentially in numbered order (1-13).
