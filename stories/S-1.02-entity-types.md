# S-1.02 — prism-core: Entity Types and State Machines

## Narrative

As a Prism platform implementer, I want all domain entity types and state machines
defined in prism-core, so that downstream crates (prism-operations, prism-mcp,
prism-audit) share a single authoritative definition of case status transitions,
alert severity levels, credential names, and cursor capacity — with formal proofs
that the constraints hold.

## Objective

Extend prism-core with domain entity types needed by the operations and MCP layers.
This story is purely additive to S-1.01; no existing code is modified. The state
machine correctness is verified by four Kani proofs (VP-005, VP-006, VP-011, VP-029).

## Tasks

1. Implement `CaseStatus` enum in `crates/prism-core/src/case.rs`:
   - Variants: `Open`, `InProgress`, `OnHold`, `Resolved`, `Closed`
   - Implement `can_transition_to(&self, next: CaseStatus) -> bool` encoding exactly
     12 valid transitions:
     - Open → InProgress, Open → OnHold
     - InProgress → OnHold, InProgress → Resolved
     - OnHold → InProgress, OnHold → Resolved
     - Resolved → Closed, Resolved → InProgress (reopen)
     - Closed → Open (reopen from closed, P1 requirement)
     - (total: count carefully — must be exactly 12 per VP-005)
   - Implement `valid_transitions(&self) -> &'static [CaseStatus]` for exhaustive
     listing (used by MCP tool for hint generation)
   - Derive: `Clone, Copy, Debug, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`

2. Implement `DispositionCode` enum in `crates/prism-core/src/case.rs`:
   - Variants: `TruePositive`, `FalsePositive`, `Benign`, `Inconclusive`, `Duplicate`,
     `TestAlert`
   - Required when transitioning to `Resolved` (enforced in prism-operations, not here)
   - Derive same traits as CaseStatus

3. Implement `AlertSeverity` enum in `crates/prism-core/src/alert.rs`:
   - Variants: `Critical`, `High`, `Medium`, `Low`, `Informational`
   - Implement `as_ocsf_severity_id(&self) -> u32` mapping to OCSF severity_id values:
     Critical → 5, High → 4, Medium → 3, Low → 2, Informational → 1
   - Derive same traits as CaseStatus

4. Implement `TimelineAnnotationType` enum in `crates/prism-core/src/case.rs`:
   - Variants: `Note`, `StatusChange`, `AlertLink`, `EvidenceLink`, `OtImpact`
   - Derive same traits

5. Implement `CredentialName` newtype in `crates/prism-core/src/credentials.rs`:
   - Inner type: `Arc<str>`
   - Constructor: `CredentialName::new(s: &str) -> Result<Self, PrismError>`
   - Reject path traversal: any string containing `/`, `\`, `..`, or null bytes (`\0`)
   - Reject empty string
   - Max length: 128 characters
   - Derive: `Clone, Debug, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`

6. Implement UUID v7 ID newtypes in `crates/prism-core/src/ids.rs`:
   - `ScheduleId(Uuid)`, `RuleId(Uuid)`, `CaseId(Uuid)`, `AlertId(Uuid)`
   - Each has a `::new() -> Self` constructor calling `Uuid::now_v7()`
   - Each has `::from_uuid(Uuid) -> Self` for deserialization
   - Derive: `Clone, Copy, Debug, PartialEq, Eq, Hash, serde::Serialize, serde::Deserialize`

7. Implement `CursorId` newtype in `crates/prism-core/src/cursor.rs`:
   - Inner type: `u64` (monotonically incrementing)
   - `CursorRegistry` struct tracking active cursors via `BTreeSet<CursorId>`
   - Constructor: `CursorRegistry::allocate(&mut self) -> Result<CursorId, PrismError>`
     - Rejects allocation when `active.len() >= 200`
     - Returns `Err(PrismError::CursorCapExceeded)` at limit
   - `CursorRegistry::release(&mut self, id: CursorId)` removes from active set
   - `CursorRegistry::active_count(&self) -> usize` for health reporting

8. Write VP-005 Kani proof in `crates/prism-core/src/proofs/case_status.rs`:
   - Enumerate all 25 `(from, to)` pairs (5 × 5); count how many return `true`
   - Assert the count equals exactly 12
   - Proof name: `proof_exactly_12_transitions`

9. Write VP-006 Kani proof in `crates/prism-core/src/proofs/case_status.rs`:
   - For every `CaseStatus` variant, assert `can_transition_to(same_variant) == false`
   - Proof name: `proof_no_self_transitions`

10. Write VP-011 Kani proof in `crates/prism-core/src/proofs/credential_name.rs`:
    - For strings containing `/`, assert `CredentialName::new()` returns `Err`
    - For strings containing `..`, assert `CredentialName::new()` returns `Err`
    - For strings containing `\0`, assert `CredentialName::new()` returns `Err`
    - Proof name: `proof_path_traversal_rejected`

11. Write VP-029 Kani proof in `crates/prism-core/src/proofs/cursor.rs`:
    - Allocate exactly 200 cursors — assert all succeed
    - Attempt 201st allocation — assert it returns `Err(CursorCapExceeded)`
    - Release one cursor; attempt allocation — assert it succeeds (cap enforces current
      count, not lifetime count)
    - Proof name: `proof_cursor_cap_200`

12. Write unit tests in `crates/prism-core/src/tests/`:
    - Test all 12 valid transitions and verify 13 invalid transitions are rejected
    - Test `AlertSeverity::as_ocsf_severity_id()` for all 5 variants
    - Test UUID v7 ID newtype creation and serde round-trip
    - Test `CursorRegistry` at boundary (199, 200, 201 allocations)

## Acceptance Criteria

**AC-1:** Given `CaseStatus::Open`, When `can_transition_to(CaseStatus::InProgress)` is
called, Then it returns `true`.

**AC-2:** Given `CaseStatus::Open`, When `can_transition_to(CaseStatus::Closed)` is
called, Then it returns `false` (Closed requires going through Resolved first).

**AC-3:** Given `CaseStatus::Resolved`, When `can_transition_to(CaseStatus::Closed)` is
called without a `DispositionCode` being set (enforced downstream), Then the state
machine itself returns `true` (disposition enforcement is in prism-operations, not here).

**AC-4:** Given a `CredentialName` input of `"../../passwd"`, When `CredentialName::new()`
is called, Then it returns `Err(PrismError::InvalidCredentialName)`.

**AC-5:** Given a `CredentialName` input containing a null byte `"key\0value"`, When
`CredentialName::new()` is called, Then it returns `Err`.

**AC-6:** Given exactly 200 active cursors in `CursorRegistry`, When a 201st cursor is
requested via `allocate()`, Then it returns `Err(PrismError::CursorCapExceeded)`.

**AC-7:** Given 200 active cursors, When one is released and `allocate()` is called
again, Then it returns `Ok(CursorId)` (cap counts active cursors, not total allocated).

**AC-8:** Given `AlertSeverity::Critical`, When `as_ocsf_severity_id()` is called,
Then it returns `5`.

**AC-9:** VP-005 Kani proof passes: exactly 12 transitions are valid.

**AC-10:** VP-006 Kani proof passes: no self-transitions exist.

**AC-11:** VP-011 Kani proof passes: path traversal patterns are all rejected.

**AC-12:** VP-029 Kani proof passes: CursorId cap of 200 is enforced.

## Architecture Compliance Rules

- All entity types MUST be `Send + Sync` (async runtime requirement)
- State machine logic MUST live in `prism-core` — not in prism-operations or prism-mcp
  (single source of truth for transition rules)
- ID newtypes MUST use UUID v7 (time-ordered, not random v4) for sorted iteration
  in RocksDB
- `CursorRegistry` MUST be a plain struct (not async) — the async wrapper lives in
  prism-query (S-3.05)
- `CredentialName` validation MUST reject all four path traversal patterns: `/`, `\`,
  `..`, null byte

## Notes

- The exact set of 12 valid transitions must be documented in a const array adjacent
  to `can_transition_to()` so the Kani proof and the runtime function share the same
  source of truth. Do not duplicate the transition list.
- UUID v7 requires the `uuid` crate with feature flag `v7`. Confirm this is in
  `Cargo.toml` for `prism-core` before implementing `ScheduleId::new()`.
- `CursorRegistry` is intentionally not async-safe at this layer. The query engine
  (S-3.05) wraps it in `Arc<Mutex<CursorRegistry>>` for concurrent access.
- `DispositionCode` is defined here but enforcement logic (requiring it on `Resolved`
  transitions) lives in prism-operations (S-4.06). This story only defines the type.
