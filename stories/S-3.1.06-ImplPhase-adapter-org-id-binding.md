# S-3.1.06-ImplPhase — prism-sensors: complete adapter OrgId binding

## Narrative

- **As a** multi-tenant sensor dispatch layer
- **I want** `OrgId` propagated structurally into every adapter constructor and
  `AdapterRegistry` indexed by `(OrgId, SensorType)` composite key
- **So that** cross-tenant adapter dispatch is a compile-time or construction-time
  impossibility, not a by-convention assumption

This story closes **F-48-H-001 (HIGH)** from pass-48 adversarial review of the Wave 3
integration gate. S-3.1.06 was declared complete in cycle-manifest E-3.1, but Task 4
("Update `init_registry` and all adapter constructors to accept `OrgId`") was not
executed. The body of `init_registry_for_org` at `crates/prism-sensors/src/lib.rs:159`
contains an explicit stub comment delegating to the legacy path, and the `_org_id`
parameter is intentionally unused (`_` prefix, line 151). Per user direction, all Wave 3
remediation work is completed before Wave 4 begins. This story is the Task 4 follow-on
for S-3.1.06.

## Objective

Replace the stub body of `init_registry_for_org` with real `OrgId` propagation. Wire
`OrgId` into each adapter constructor and store it as a typed field. Rekey
`AdapterRegistry` by `(OrgId, SensorType)`. Add a new `SensorError::OrgIdMismatch`
variant for runtime cross-org guard detection. Deprecate legacy `init_registry`.
Migrate all existing test callers to the new constructor signatures.

## Acceptance Criteria

**AC-001 — Structural OrgId binding in all adapter constructors (traces to BC-3.1.001
postcondition 1 and BC-3.2.001 precondition 4):**
`CrowdStrikeAdapter::new`, `CyberintAdapter::new`, `ClarotyAdapter::new`, and
`ArmisAdapter::new` each accept `org_id: OrgId` as their first parameter and store it
in a typed `org_id: OrgId` field on the struct. The `_` prefix on the `init_registry_for_org`
parameter is removed. After this change, `grep -n "_org_id" crates/prism-sensors/src/`
returns zero hits.

**AC-002 — AdapterRegistry indexed by `(OrgId, SensorType)` composite key (traces to
BC-3.2.001 invariant 1 and BC-3.1.003 invariant 2):**
`AdapterRegistry` internal `HashMap` is retyped from `HashMap<SensorType, Arc<dyn
SensorAdapter>>` to `HashMap<(OrgId, SensorType), Arc<dyn SensorAdapter>>`. `register`,
`get`, `len`, and `is_empty` are updated accordingly. A test that registers adapters for
two distinct `OrgId` values with the same `SensorType` confirms that `get(org_id_A,
SensorType::CrowdStrike)` and `get(org_id_B, SensorType::CrowdStrike)` return different
adapter instances (different `Arc` pointer addresses).

**AC-003 — `init_registry_for_org` stub body replaced with real propagation (traces to
BC-3.2.001 precondition 4):**
The stub comment `// Stub: delegates to legacy init_registry until S-3.1.06 implementation
wires org_id into each adapter constructor.` is removed. The function body constructs
each adapter via `Adapter::new(org_id, ...)` and calls `registry.register_for_org(org_id,
Arc::new(adapter))` (or equivalent API). A unit test verifies that `init_registry_for_org`
with `org_id = A` produces a registry where `registry.get(A, SensorType::CrowdStrike)` is
`Some` and `registry.get(B, SensorType::CrowdStrike)` is `None` for any `B ≠ A`.

**AC-004 — `SensorError::OrgIdMismatch` typed error variant for cross-org dispatch guard
(traces to BC-3.2.001 EC-003, NEW error code — add to error-taxonomy.md):**
A new `SensorError::OrgIdMismatch` variant is added with error code `E-SENSOR-060`:
```
#[error("E-SENSOR-060: OrgId mismatch: adapter registered for {adapter_org_id} received query for {query_org_id}")]
OrgIdMismatch { adapter_org_id: OrgId, query_org_id: OrgId }
```
This variant's `is_transient()` returns `false` — it is a permanent dispatch error. A
unit test verifies that a dispatch attempt where `SensorSpec.org_id != adapter.org_id`
returns `Err(SensorError::OrgIdMismatch { .. })` rather than silently proceeding. The
error code `E-SENSOR-060` must be added to `.factory/specs/prd-supplements/error-taxonomy.md`
in the same PR (flag if out of scope for this story).

**AC-005 — Legacy `init_registry` marked `#[deprecated]` and retained for one wave
(traces to BC-3.1.001 invariant 1 — org identity resolution must be available to all
callers during migration window):**
`init_registry` retains its `#[deprecated(since = "0.2.0", note = "use
init_registry_for_org(org_id, ...) instead (S-3.1.06)")]` attribute (already present at
`lib.rs:106`). Confirm the attribute is present and that `cargo build --workspace`
produces deprecation warnings for any caller that invokes it. Removal is deferred to
Wave 5 (after all callers have migrated).

**AC-006 — Downstream test callers migrated to new constructor signatures (traces to
BC-3.1.003 invariant 1 — bijectivity holds at all times, so test callers must provide a
valid OrgId):**
All call sites of `ArmisAdapter::new`, `ClarotyAdapter::new`, `CrowdStrikeAdapter::new`,
and `CyberintAdapter::new` in `crates/prism-sensors/tests/` are updated to pass
`DEFAULT_ORG_ID_BYTES`-derived `OrgId` as first argument. `cargo test -p prism-sensors`
passes with zero `E0061` (wrong number of arguments) errors.

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Org A's adapter is retrieved with Org B's `SensorType` key from registry | `registry.get((org_id_B, SensorType::CrowdStrike))` returns `None` — Org A's adapter is not visible to Org B callers |

## Purity Classification

| Module | Classification | Justification |
|--------|----------------|---------------|
| `SensorSpec.org_id` field | Pure | Struct field; no I/O; already present with `#[serde(default)]` |
| Adapter struct `org_id` fields (`CrowdStrikeAdapter`, `CyberintAdapter`, `ClarotyAdapter`, `ArmisAdapter`) | Pure (field addition) | Typed field added at construction; no I/O during field storage |
| Adapter `new()` constructors | Effectful | HTTP client construction involves network setup; `org_id` parameter is stored but does not trigger I/O |
| `SensorError::OrgIdMismatch` guard in `fetch()` | Pure (guard path) | Early return before any network call; pure type comparison |
| `AdapterRegistry` rekey | Pure | HashMap is an in-memory data structure; `register` and `get` are pure read/write operations on owned state |
| `init_registry_for_org` body | Effectful | Calls adapter constructors (effectful); builds registry in memory |
| New test file `org_id_binding.rs` | Pure (assertions) | All assertions operate on in-memory adapter instances constructed in test setup |

## Forbidden Dependencies

The following modules MUST NOT appear as new imports in `crates/prism-sensors/src/`:

- `prism-core::org_registry::OrgRegistry` — `prism-sensors` does not resolve slugs; it
  receives an already-resolved `OrgId` from the dispatch layer above it (ADR-006 §2.3).
  If `OrgRegistry` appears as an import in any `prism-sensors/src/` file after this
  change, the build MUST fail (add a `#[deny(unused_imports)]` or CI grep check).
- Any async runtime other than `tokio` — the existing runtime choice is fixed
  (AD-013).
- `proptest` in non-test code — proptest is a `[dev-dependencies]` only.
