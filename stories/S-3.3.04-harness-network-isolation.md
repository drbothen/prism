# S-3.3.04 — prism-dtu-harness: network isolation mode (per-port, real HTTP)

## Narrative

As a Prism integration-test author, I want `IsolationMode::Network` in
`prism-dtu-harness` where each `(OrgId, DtuType)` pair gets its own TCP listener
with real HTTP routing, so that cross-org credential routing bugs — where a request
bearing OrgA's token is sent to OrgB's endpoint — are caught as observable HTTP 401
failures rather than silent data leaks.

## Objective

Extend `prism-dtu-harness` with `IsolationMode::Network`. Each `(OrgId, DtuType)`
receives its own OS-assigned TCP port. All listeners are pre-allocated simultaneously
before any `start_on` call (no retry-on-EADDRINUSE, locked per ADR-011 D-058). The
`customer_endpoints: HashMap<(OrgId, DtuType), SocketAddr>` table is populated
atomically during `build()` and is immutable for the harness lifetime. Cleanup on
`drop` releases all ports and kills child tasks.

## Tasks

1. Extend `src/types.rs` — add `IsolationMode::Network` variant (S-3.3.03 added `Logical`).

2. Extend `src/builder.rs` — `build()` dispatch on `IsolationMode`:
   - `Logical` path: existing from S-3.3.03
   - `Network` path:
     a. For each `(OrgId, DtuType)`: bind `TcpListener::bind("127.0.0.1:0")` — collect ALL listeners before any `start_on`
     b. Store `(OrgKey, listener)` pairs
     c. `tokio::join!` over all `clone.start_on(listener_addr, shutdown_rx, None)` calls
     d. Total `build()` MUST complete within 5s (BC-3.5.002 postcondition 5)
     e. On `TcpListener::bind` failure: return `Err(HarnessError::PortConflict { ... })`; no partial harness
     f. If OS cannot allocate all ports: `Err(HarnessError::PortExhausted)`

3. Expose `customer_endpoints(&self) -> &HashMap<OrgKey, SocketAddr>` as the primary accessor name for network mode (alias for `endpoints` — consistent API surface).

4. Each clone in network mode is initialized with its own `admin_token` (per ADR-003 Amendment §5); the token is passed at `with_customer_overrides` time and stored per `OrgKey`.

5. Cross-org credential mismatch: test client routed to wrong endpoint must receive HTTP 401 from the target clone's auth middleware — not a silent data response.

6. Create `tests/network_isolation_test.rs` implementing BC-3.5.002 canonical test vectors TV-1 through TV-7.

## Acceptance Criteria

**AC-001:** For a 3-org × 4-sensor (12-clone) network-mode harness, `customer_endpoints` after `build()` contains exactly 12 entries with pairwise-distinct `SocketAddr` values (no two clones share a port).
(traces to BC-3.5.002 postcondition 4; invariant 1; TV-4)

**AC-002:** A Prism MCP request bearing `OrgId(A)` credentials routed to `OrgId(B)`'s endpoint returns HTTP 401 — not HTTP 200 and not a silent empty result.
(traces to BC-3.5.002 postcondition 2; TV-3; EC-001)

**AC-003:** For all pairs `(OrgA, OrgB)` where `OrgA ≠ OrgB`, the device ID sets returned by per-org queries are pairwise disjoint.
(traces to BC-3.5.002 postcondition 1; TV-1)

**AC-004:** A request to an `(OrgId, DtuType)` pair not in `customer_endpoints` returns `HarnessError::UnknownOrg`; no request is forwarded to any live clone.
(traces to BC-3.5.002 postcondition 3; TV-7; EC-005)

**AC-005:** The entire network-mode harness — all clones — completes `build().await` within 5 seconds total on a CI runner (parallel startup via `tokio::join!`).
(traces to BC-3.5.002 postcondition 5; invariant 2)

**AC-006:** After `drop(harness)`, all TCP listeners bound during the harness lifetime are released; `TcpStream::connect` to every previously-recorded clone address returns `ConnectionRefused` within 1 second.
(traces to BC-3.5.002 postcondition 6; TV-6)

**AC-007:** Port pre-allocation binds all listeners simultaneously before any `start_on` call; there is no retry-on-EADDRINUSE loop anywhere in the network-mode build path.
(traces to BC-3.5.002 precondition 4; invariant 2)

**AC-008:** Two parallel test runs with `--test-threads=4` each receive independent OS-assigned port sets; no `EADDRINUSE` between concurrent runs.
(traces to BC-3.5.002 EC-002)

## Architecture Compliance Rules (MANDATORY)

| Rule | Source | Enforcement |
|------|--------|-------------|
| Simultaneous port bind before `start_on` | ADR-011 §2.5; D-058 | All `TcpListener::bind` calls precede all `start_on` calls |
| No EADDRINUSE retry | ADR-011 D-058 locked | No retry loop in bind phase |
| `customer_endpoints` immutable after `build()` | BC-3.5.002 invariant 1 | Field is `HashMap` inside `Harness`; no `&mut` accessor exposed |
| Network-mode uses same `BehavioralClone::start_on` path | BC-3.5.002 invariant 4 | No code duplication; mode difference is in test client routing only |
