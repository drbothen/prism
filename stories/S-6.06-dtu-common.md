# S-6.06 — prism-dtu-common: DTU Common Infrastructure

## Narrative

As a Prism integration-test infrastructure author, I want a shared `prism-dtu-common`
crate that provides the `BehavioralClone` trait, configurable latency and failure
injection middleware, a fixture loader, a generic `SyslogReceiver`, a generic
`WebhookReceiver`, and shared assertion utilities, so that the 13 per-surface DTU
crates (S-6.07 through S-6.19) can be built on a consistent foundation and integration
tests can manage the DTU fleet uniformly.

## Objective

Implement `prism-dtu-common` — the shared test infrastructure crate. This crate provides
ONLY the building blocks used by all 13 per-surface DTU clones. It does NOT implement
any per-sensor or per-service behavioral clone. Scope: `BehavioralClone` trait
(`start()`, `reset()`, `configure()`, `bound_addr()`), `LatencyLayer` tower middleware,
`FailureLayer` tower middleware, fixture loader (TOML/JSON fixture files from `fixtures/`
directories), `SyslogReceiver` (RFC 5424, UDP + TCP), `WebhookReceiver` (generic HTTP
POST capture), and shared test utilities (fidelity validator hooks, response capture,
assertion helpers). All 13 per-surface DTU crates depend on this crate as a
dev-dependency.

## Delivery Model — Design Decision

**In-process Axum server per DTU surface** — launched in a `tokio::task::spawn`, bound
to `127.0.0.1:0` (ephemeral port), managed via the `BehavioralClone` trait. See
`dtu-assessment.md §4` for full rationale. `prism-dtu-common` provides the framework;
per-surface crates provide the route handlers.

## Tasks

1. Create `crates/prism-dtu-common/Cargo.toml`:
   - `[lib]` crate type; this is a dev-dependency of all 13 per-surface DTU crates
   - Gated via `#[cfg(any(test, feature = "dtu"))]` — never compiled into production binary
   - Dependencies: `axum` (0.7.x), `tokio` (1.x full), `tower` (0.4.x), `tower-http`,
     `serde` (1.x derive), `serde_json` (1.x), `rand` (0.8.x), `rand_chacha` (0.3.x),
     `http` (1.x), `async-trait` (0.1.x), `tracing` (0.1.x), `tokio-util`
   - No production prism crate dependencies except `prism-core` type aliases if needed

2. Create `crates/prism-dtu-common/src/lib.rs`:
   - Re-export: `BehavioralClone`, `StubConfig`, `FailureMode`, `SeededRng`,
     `LatencyLayer`, `FailureLayer`, `fixture_loader`, `SyslogReceiver`,
     `WebhookReceiver`, `FidelityValidator`, and all test utility types

3. Create `crates/prism-dtu-common/src/clone.rs` — `BehavioralClone` trait:
   ```rust
   #[async_trait]
   pub trait BehavioralClone: Send + Sync + 'static {
       async fn start(&mut self) -> anyhow::Result<()>;
       async fn reset(&self) -> anyhow::Result<()>;
       async fn configure(&self, config: serde_json::Value) -> anyhow::Result<()>;
       fn bound_addr(&self) -> std::net::SocketAddr;
       fn base_url(&self) -> String { format!("http://{}", self.bound_addr()) }
   }
   ```

4. Create `crates/prism-dtu-common/src/config.rs`:
   - `StubConfig { seed: u64, latency_ms: u64, failure_mode: FailureMode }`
   - `FailureMode` enum: `None`, `RateLimit { after_n_requests: u32, retry_after_secs: u32 }`,
     `InternalError { at_request_n: u32 }`, `NetworkTimeout { after_ms: u64 }`,
     `AuthReject` (causes 401 on all authenticated endpoints)
   - `impl Default for StubConfig`: `seed: 42, latency_ms: 0, failure_mode: None`

5. Create `crates/prism-dtu-common/src/seed.rs`:
   - `pub fn seeded_rng(seed: u64) -> rand_chacha::ChaCha20Rng`
   - Same seed → same sequence, always. Forbidden: `rand::thread_rng()`.

6. Create `crates/prism-dtu-common/src/layers/mod.rs` + `latency.rs` + `failure.rs`:
   - `LatencyLayer`: `tower::Layer` impl; sleeps `config.latency_ms` ms before forwarding
   - `FailureLayer`: `tower::Layer` impl; uses `Arc<AtomicU32>` request counter; when
     `FailureMode::RateLimit` threshold is crossed, returns HTTP 429 with `Retry-After`
     header; when `FailureMode::InternalError` fires, returns HTTP 500; when
     `FailureMode::AuthReject` is active, returns HTTP 401 on all requests

7. Create `crates/prism-dtu-common/src/fixture.rs`:
   - `pub fn load_fixture(crate_dir: &str, name: &str) -> serde_json::Value`
   - Reads `{crate_dir}/fixtures/{name}.json`; panics with clear message if absent
   - `pub fn load_fixture_as<T: DeserializeOwned>(crate_dir: &str, name: &str) -> T`
   - Fixture files are checked in to each per-surface crate's `fixtures/` directory

8. Create `crates/prism-dtu-common/src/syslog.rs` — `SyslogReceiver`:
   - Binds UDP + TCP socket on `127.0.0.1:0` (ephemeral)
   - Receives RFC 5424 syslog messages; stores in `Arc<Mutex<Vec<String>>>`
   - `pub fn received_messages(&self) -> Vec<String>` for assertion in integration tests
   - `pub fn reset(&self)` clears the capture store
   - Used by action syslog tests AND log-forward syslog tests — no per-surface crate needed

9. Create `crates/prism-dtu-common/src/webhook.rs` — `WebhookReceiver`:
   - Axum server bound to `127.0.0.1:0`; handles POST to any path
   - Stores raw body bytes + path + headers in `Arc<Mutex<Vec<CapturedRequest>>>`
   - `pub struct CapturedRequest { pub path: String, pub body: Bytes, pub headers: HeaderMap }`
   - `pub fn received_payloads(&self) -> Vec<CapturedRequest>` for assertions
   - `pub fn reset(&self)` clears the capture store
   - Used by webhook action tests AND generic webhook forwarder tests

10. Create `crates/prism-dtu-common/src/fidelity.rs` — `FidelityValidator`:
    - `FidelityValidator::run(base_url: &str, checks: Vec<FidelityCheck>) -> FidelityReport`
    - `FidelityCheck { endpoint: String, method: Method, body: Option<Value>, expected_status: u16, required_fields: Vec<String> }`
    - `FidelityReport { checks_passed: usize, checks_failed: usize, failures: Vec<FidelityFailure> }`
    - `FidelityFailure { endpoint: String, reason: String }`
    - Intended for use in `#[test] fn dtu_fidelity_check()` inside each per-surface crate

11. Create `crates/prism-dtu-common/src/test_utils.rs`:
    - `assert_status(response: &Response, expected: StatusCode)` — panic with diff on mismatch
    - `assert_field_present(body: &Value, field: &str)` — panic if field absent
    - `assert_header_present(response: &Response, header: &str)` — panic if header absent
    - `build_test_client() -> reqwest::Client` — preconfigured client (no redirects, 5s timeout)

12. Confirm the `integration-test` and `dtu-validate` targets exist in `Justfile`
    (defined in S-0.02). If S-0.02 is not yet implemented, add a note in the PR that
    these targets must be added. Do NOT modify the Justfile in this story if S-0.02 is
    delivered in parallel.

## Acceptance Criteria

**AC-1:** Given `BehavioralClone::start()` is called on a per-surface DTU instance
that depends on `prism-dtu-common`, When the server binds, Then `bound_addr()` returns
a valid `SocketAddr` and the server is reachable at `http://127.0.0.1:{port}`.

**AC-2:** Given `FailureLayer` is configured with `FailureMode::RateLimit { after_n_requests: 5, retry_after_secs: 60 }`,
When the 6th HTTP request arrives, Then the layer returns HTTP 429 with header
`Retry-After: 60` without forwarding to the route handler.

**AC-3:** Given `FailureLayer` is configured with `FailureMode::AuthReject`,
When any HTTP request arrives, Then the layer returns HTTP 401 regardless of the
`Authorization` header value.

**AC-4:** Given `LatencyLayer` is configured with `latency_ms: 100`, When a request
is processed, Then the response is delayed by at least 80ms (allowing for timer
resolution) before being returned.

**AC-5:** Given `seeded_rng(42)` is called twice, When any random value sequence
is drawn from each RNG, Then both sequences are identical (deterministic, same seed).

**AC-6:** Given `SyslogReceiver` is started, When a well-formed RFC 5424 syslog
message is sent to its UDP port, Then `received_messages()` returns a list containing
that message.

**AC-7:** Given `WebhookReceiver` is started, When a POST request with JSON body
is sent to its bound address, Then `received_payloads()` returns a `CapturedRequest`
with the correct path and body bytes.

**AC-8:** Given `FidelityValidator::run()` is called with a check expecting field
`"status"` in the response body, When the DTU response body does not contain `"status"`,
Then the `FidelityReport` contains a `FidelityFailure` for that check.

**AC-9:** Given `load_fixture("prism-dtu-crowdstrike", "devices-page1")` is called,
When the file `crates/prism-dtu-crowdstrike/fixtures/devices-page1.json` exists,
Then the function returns the parsed JSON value without error.

## Architecture Compliance Rules

- `prism-dtu-common` MUST be gated behind `#[cfg(any(test, feature = "dtu"))]` — never
  compiled into production binaries. Same rule applies to all 13 per-surface crates.
- All DTU servers MUST bind to `127.0.0.1:0` (ephemeral port) via the `BehavioralClone`
  contract — never a hardcoded port (parallel test execution safety).
- `seeded_rng()` MUST use `ChaCha20Rng`; use of `rand::thread_rng()` anywhere in
  `prism-dtu-common` or per-surface crates is forbidden.
- `prism-dtu-common` MUST NOT depend on production Prism crates beyond `prism-core`
  type aliases — no circular dev-dependency chains.
- `SyslogReceiver` and `WebhookReceiver` MUST be in `prism-dtu-common`, not replicated
  in per-surface crates. Per-surface crates depend on common; common does not depend
  on any per-surface crate.
- The `BehavioralClone` trait defines the only interface contract for fleet management.
  Per-surface crates MUST implement it, not invent alternative management APIs.

**Forbidden dependencies:** `prism-dtu-common` MUST NOT depend on `prism-sensors`,
`prism-query`, `prism-operations`, `prism-mcp`, `prism-spec-engine`, or `prism-audit`.
These are production crates; the DTU must not create a circular dependency. If a build
adds any of these to `prism-dtu-common/Cargo.toml`, the build MUST fail via deny rule.

## Risk Mitigations

### R-DTU-005: Maintenance burden on API evolution

**Risk:** Vendors evolve their APIs. CrowdStrike regularly releases new API versions;
Armis has versioned their API. Without a process for DTU maintenance, the DTU drifts
from the real API and integration tests begin to test an outdated contract.

**Mitigation (as implemented in prism-dtu-common):** The `FidelityValidator`
(`src/fidelity.rs`) runs an 85%-fidelity-threshold CI job (`just dtu-validate`) that
catches schema drift when fixtures are updated. A team policy requiring DTU fixture
updates when sensor TOML specs are updated closes the loop. This policy must be
documented as a contribution rule in `CONTRIBUTING.md` for the Prism repository. The
fidelity validator hooks in `src/fidelity.rs` are the shared enforcement mechanism
that all 13 per-surface DTU crates inherit from this story. See `dtu-assessment.md §10
R-DTU-005` for the full risk statement.
