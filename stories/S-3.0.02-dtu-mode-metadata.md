# S-3.0.02 — prism-core: register DTU_DEFAULT_MODE registry (10-entry DtuRegistryEntry slice) per ADR-007 §2.3

## Narrative
- **As a** config validator (introduced in S-3.3.01)
- **I want** a single centralized `DTU_DEFAULT_MODE: &[DtuRegistryEntry]` static in `prism-core`
  that maps every known DTU type string to its default `DtuMode` and `test_only` flag
- **So that** the startup config validator can enforce ADR-007 §2.4 rules (Security Telemetry types
  must be `client`; test-only types are rejected in production configs) without scattering
  classification knowledge across individual DTU crates or hardcoding it in the validator

## Description

ADR-007 §2.3 specifies a **centralized** compile-time registry in `prism-core` (co-located with
`OrgRegistry` per D-047), not per-crate declarations. The rationale section of ADR-007 explicitly
rejects per-crate constants:

> "A per-crate declaration (e.g., a `const DEFAULT_MODE: DtuMode` in each crate's `lib.rs`) would
> scatter the classification across 11 crates and require reading 11 files to audit the full
> isolation posture. A centralized registry in `prism-core` makes the full classification visible
> in one place, auditable in one grep, and enforceable in one validation function."

This story implements:
1. `DtuRegistryEntry` struct (`type_name`, `default_mode`, `test_only` fields) in `prism-core`
2. `DTU_DEFAULT_MODE: &[DtuRegistryEntry]` static with all 10 known DTU types

The 10 registry entries per ADR-007 §2.3:

| type_name | default_mode | test_only |
|-----------|-------------|-----------|
| claroty | DtuMode::Client | false |
| armis | DtuMode::Client | false |
| crowdstrike | DtuMode::Client | false |
| cyberint | DtuMode::Client | false |
| demo-server | DtuMode::Client | true |
| slack | DtuMode::Shared | false |
| pagerduty | DtuMode::Shared | false |
| jira | DtuMode::Shared | false |
| nvd | DtuMode::Shared | false |
| threatintel | DtuMode::Shared | false |

`demo-server` is present in the registry with `test_only: true` so the validator can emit
`E-CFG-013` (test-only type in production config) rather than `E-CFG-004` (unknown type) — a
distinct error code per D-051.

The `DtuMode` enum (`Shared` | `Client`) also lives in `prism-core` alongside `DtuRegistryEntry`.
Individual DTU crates do NOT define their own `DTU_DEFAULT_MODE` constants.

## Acceptance Criteria

**AC-1 (traces to BC-3.2.005 precondition 3):** `DtuMode` is defined in `prism-core` as
`#[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Deserialize)] #[serde(rename_all = "lowercase")]
pub enum DtuMode { Shared, Client }`. The enum compiles on stable Rust.

**AC-2 (traces to BC-3.2.005 invariant 1):** `DtuMode` derives `Debug, Clone, Copy, PartialEq, Eq`
— it is a value type with no interior mutability.

**AC-3 (traces to BC-3.2.005 precondition 3):** `DtuRegistryEntry` is defined in `prism-core` with
fields `type_name: &'static str`, `default_mode: DtuMode`, and `test_only: bool`.

**AC-4 (traces to BC-3.2.005 postcondition 1):** `DTU_DEFAULT_MODE` is a
`pub static DTU_DEFAULT_MODE: &[DtuRegistryEntry]` in `prism-core` containing exactly 10 entries
(the 9 production types + `demo-server`).

**AC-5 (traces to BC-3.2.005 postcondition 1):** The 5 MSSP Coordination entries (`slack`,
`pagerduty`, `jira`, `nvd`, `threatintel`) have `default_mode: DtuMode::Shared` and
`test_only: false`.

**AC-6 (traces to BC-3.2.005 postcondition 2):** The 4 Security Telemetry entries (`claroty`,
`armis`, `crowdstrike`, `cyberint`) have `default_mode: DtuMode::Client` and `test_only: false`.

**AC-7 (traces to BC-3.2.005 invariant 1):** The `demo-server` entry has `default_mode:
DtuMode::Client` and `test_only: true` per D-051. A doc comment on the entry states:
`// D-051: test infrastructure only — excluded from production-allowed set by validator`.

**AC-8 (traces to BC-3.2.005 postcondition 4):** No individual DTU crate (`prism-dtu-claroty`,
`prism-dtu-armis`, etc.) exports a `DTU_DEFAULT_MODE` constant or a `dtu_default_mode()` function.
Classification lives exclusively in `prism-core`.

## Architecture Compliance Rules

| Rule | Source | Enforcement |
|------|--------|-------------|
| `DTU_DEFAULT_MODE` lives in `prism-core`, NOT in individual DTU crates | ADR-007 §2.3 Rationale | Code review; grep for `DTU_DEFAULT_MODE` in any `prism-dtu-*` crate fails CI |
| `DtuMode` enum is `const`-compatible (Copy + no heap allocation) | BC-3.2.005 invariant 1 | Rust type system; `Copy` derives prevents accidental heap usage |
| `demo-server` entry carries `test_only: true` | D-051 | Unit test asserting `test_only == true` for `"demo-server"` entry |
| No per-crate `dtu_default_mode()` method or constant | ADR-007 §2.3 Rationale | grep CI check: `pub fn dtu_default_mode\|pub const DTU_DEFAULT_MODE` in `prism-dtu-*` = fail |
| No runtime setter for `DtuMode` | BC-3.2.005 invariant 2 | grep for `set_mode`, `with_mode` methods |
