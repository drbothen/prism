# S-3.0.02 — prism-dtu-*: add dtu_default_mode() to existing 7 DTU crates

## Narrative
- **As a** config validator (introduced in S-3.3.01)
- **I want** each DTU crate to expose a `dtu_default_mode()` constant that declares
  whether it is `"shared"` or `"client"` by nature
- **So that** the startup config validation can enforce ADR-007 §2.4 rule 3 (Security
  Telemetry types must be `client`; MSSP Coordination types default to `shared`)
  without hardcoding the type registry in the validator

## Description

ADR-007 §2.3 defines a per-DTU-type default mode registry. This story adds the
compile-time metadata to the 7 existing DTU crates (excluding `prism-dtu-demo-server`,
which is test infrastructure only and NOT in the default registry per D-051). The metadata
is a `pub const fn dtu_default_mode() -> DtuMode` function (or equivalent `pub const
DTU_DEFAULT_MODE: DtuMode` constant) in each crate's `lib.rs`. The config validator in
S-3.3.01 reads this constant to enforce mode compatibility at startup.

**Mode assignments per ADR-007 §2.1 / D-042:**
- `"shared"` (MSSP Coordination): `prism-dtu-slack`, `prism-dtu-pagerduty`, `prism-dtu-jira`,
  `prism-dtu-nvd`, `prism-dtu-threatintel`
- `"client"` (Security Telemetry): `prism-dtu-claroty`, `prism-dtu-armis`,
  `prism-dtu-crowdstrike`, `prism-dtu-cyberint`

`prism-dtu-demo-server` is intentionally excluded (D-051: test infrastructure only;
NOT in the default mode registry).

## Acceptance Criteria

**AC-1 (traces to BC-3.2.005 precondition 3):** Each of the 9 DTU crates exposes a
`pub const DTU_DEFAULT_MODE: DtuMode` (or equivalent `pub const fn dtu_default_mode()
-> DtuMode`) that compiles without error on stable Rust.

**AC-2 (traces to BC-3.2.005 postcondition 1):** The 5 MSSP Coordination crates
(`slack`, `pagerduty`, `jira`, `nvd`, `threatintel`) return `DtuMode::Shared`.

**AC-3 (traces to BC-3.2.005 postcondition 2):** The 4 Security Telemetry crates
(`claroty`, `armis`, `crowdstrike`, `cyberint`) return `DtuMode::Client`.

**AC-4 (traces to BC-3.2.005 invariant 1):** `DtuMode` is `#[derive(Debug, Clone, Copy,
PartialEq, Eq)]` in `prism-dtu-common` — it is a value type with no interior mutability.
`DTU_DEFAULT_MODE` is a `const` (not `static mut`), enforcing immutability at the
type level.

**AC-5 (traces to BC-3.2.005 invariant 4):** `prism-dtu-demo-server` does NOT export
`DTU_DEFAULT_MODE` or `dtu_default_mode()` — it is not part of the default registry per
D-051. A compile-time comment documents this exclusion.

## Edge Cases

| ID | Scenario | Expected Behavior |
|----|----------|-------------------|
| EC-001 | Config validator calls `DTU_DEFAULT_MODE` for a Security Telemetry crate but config has `mode = "shared"` | Validator returns startup error per BC-3.2.005 EC-001; this story only provides the constant, not the validator logic (that is S-3.3.01) |
| EC-002 | New DTU crate added in a future wave without `DTU_DEFAULT_MODE` | Compile error in the config validator (which calls the constant); enforced structurally |
| EC-003 | `prism-dtu-demo-server` referenced in config | Demo server has no `DTU_DEFAULT_MODE`; S-3.3.01 handles this by rejecting demo-server from the default registry at config parse time |

   ```rust
   #[derive(Debug, Clone, Copy, PartialEq, Eq, serde::Deserialize)]
   #[serde(rename_all = "lowercase")]
   pub enum DtuMode { Shared, Client }
   ```

3. [ ] For each of the 9 production DTU crates, add to `src/lib.rs`:
   ```rust
   use prism_dtu_common::DtuMode;
   pub const DTU_DEFAULT_MODE: DtuMode = DtuMode::<Shared|Client>;
   ```
4. [ ] Add a doc comment on each constant stating the ADR-007 §2.1 rationale
   (e.g., `/// MSSP Coordination tool — shared mode by default (ADR-007 §2.1)`).
5. [ ] Confirm `prism-dtu-demo-server/src/lib.rs` does NOT export `DTU_DEFAULT_MODE`;
   add a `// D-051: demo-server is test infrastructure only — not in default registry`
   comment.

   `assert_eq!(prism_dtu_slack::DTU_DEFAULT_MODE, DtuMode::Shared)`.
7. [ ] `cargo test --workspace` — all tests pass.
