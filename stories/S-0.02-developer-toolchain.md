# S-0.02 — devops: Developer Toolchain Bootstrap

## Narrative

As a Prism contributor, I want a single `just check` command that runs the identical PR
gate locally, pre-commit hooks that auto-format and lint changed files on commit, and a
`dev-setup.sh` script that installs all required cargo tools in one step, so that I
catch quality issues before pushing rather than waiting for CI feedback.

## Objective

Implement the developer toolchain bootstrap: a `Justfile` with standard targets, a
`lefthook.yml` pre-commit hook configuration, toolchain configuration files
(`kani.toml`, `rustfmt.toml`, `clippy.toml`, `deny.toml`, `rust-toolchain.toml`), a
`.semgrep/` directory with Rust SAST rules for credential handling and unsafe patterns,
and a `scripts/dev-setup.sh` that installs all cargo toolchain extensions. Together these
give every contributor a local environment that matches CI identically.

## Tasks

1. Create `rust-toolchain.toml` at workspace root:
   ```toml
   [toolchain]
   channel = "stable"
   components = ["rustfmt", "clippy", "rust-src"]
   targets = [
     "aarch64-apple-darwin",
     "x86_64-apple-darwin",
     "x86_64-unknown-linux-gnu",
     "x86_64-unknown-linux-musl",
     "x86_64-pc-windows-msvc",
   ]
   ```
   - Pins the stable channel; `rustup` uses this file automatically
   - `rust-src` is required for Kani

2. Create `rustfmt.toml` at workspace root:
   ```toml
   edition = "2021"
   max_width = 100
   use_small_heuristics = "Default"
   imports_granularity = "Crate"
   group_imports = "StdExternalCrate"
   ```

3. Create `clippy.toml` at workspace root:
   ```toml
   cognitive-complexity-threshold = 30
   too-many-arguments-threshold = 8
   ```
   - `await_holding_lock = DENY` is applied via workspace-level `[lints.clippy]` in
     root `Cargo.toml` (not `clippy.toml`) — add to root Cargo.toml `[workspace.lints]`
     section:
     ```toml
     [workspace.lints.clippy]
     await_holding_lock = "deny"
     unwrap_used = "warn"
     ```

4. Create `kani.toml` at workspace root matching `tooling-selection.md` § Kani
   Configuration exactly:
   ```toml
   [general]
   default-unwind = 10

   [verification]
   timeout = 300
   memory-limit = 8192
   ```

5. Create `deny.toml` at workspace root:
   - `[licenses]` section: allowlist of OSI-approved licenses used by this project
     (MIT, Apache-2.0, BSD-2-Clause, BSD-3-Clause, ISC, Unicode-DFS-2016, Zlib)
   - `[advisories]` section: `db-path = "~/.cargo/advisory-db"`, `db-urls = ["..."]`,
     `vulnerability = "deny"`, `unmaintained = "warn"`, `yanked = "warn"`
   - `[bans]` section: `multiple-versions = "warn"`, `wildcards = "deny"` (no wildcard
     version requirements in Cargo.toml files)
   - `[sources]` section: `unknown-registry = "deny"`, `unknown-git = "deny"` (only
     crates.io and the workspace itself are permitted sources)

6. Create `Justfile` at workspace root with the following targets:
   ```just
   # Default: show available targets
   default:
       @just --list

   # Run the full test suite (identical to CI)
   test:
       cargo test --workspace

   # Run the PR gate locally (identical to CI step order)
   check:
       cargo fmt --check
       cargo clippy -- -D warnings
       cargo test --workspace
       cargo deny check
       cargo audit
       cargo semver-checks

   # Format all code
   fmt:
       cargo fmt

   # Run clippy with warnings
   clippy:
       cargo clippy -- -D warnings

   # Generate coverage report (requires cargo-llvm-cov)
   cov:
       cargo llvm-cov --workspace --html

   # Run mutation testing (requires cargo-mutants)
   mutants:
       cargo mutants --workspace

   # Run a specific fuzz target locally (requires cargo-fuzz)
   # Usage: just fuzz-local prism-query fuzz_prismql_parser
   fuzz-local crate target:
       cargo fuzz run --manifest-path crates/{{crate}}/fuzz/Cargo.toml {{target}} -- -max_total_time=60

   # Run Kani proofs locally (requires kani-verifier installed)
   kani-local:
       cargo kani --workspace --timeout 300 --mem-limit 8192

   # Clean build artifacts
   clean:
       cargo clean

   # Check for unused dependencies (requires cargo-udeps + nightly)
   udeps:
       cargo +nightly udeps --workspace

   # Run integration tests against DTU sensor stubs
   # NOTE: Requires S-6.06 (prism-dtu crate) to be implemented first.
   # Until S-6.06 lands, this target will fail with "feature `dtu` not found" — this is expected.
   integration-test:
       cargo test --workspace --features dtu -- --test-threads=1 integration

   # Start DTU sensor stubs standalone for manual development use
   # NOTE: Requires S-6.06 (prism-dtu crate). Will fail until S-6.06 is implemented.
   dtu-start:
       cargo run --features dtu --bin dtu-runner

   # Run DTU fidelity validation against the DTU fleet
   # NOTE: Requires S-6.06. Will fail until S-6.06 is implemented.
   dtu-validate:
       cargo test --workspace --features dtu -- --test-threads=1 dtu_fidelity

   # Install all development toolchain extensions (idempotent)
   setup:
       ./scripts/dev-setup.sh
   ```

7. Create `lefthook.yml` at workspace root:
   ```yaml
   pre-commit:
     parallel: false
     commands:
       fmt:
         run: cargo fmt -- {staged_files}
         glob: "*.rs"
         stage_fixed: true
       clippy:
         run: cargo clippy --quiet -- -D warnings
         glob: "*.rs"
   ```
   - `stage_fixed: true` re-stages files that `cargo fmt` modified
   - `{staged_files}` is a lefthook substitution that passes only the changed `.rs`
     files to `cargo fmt`, avoiding reformatting the entire workspace on each commit

8. Create `.semgrep/` directory with two Rust SAST rule files:
   - `.semgrep/credential-handling.yml`:
     - Rule `prism-no-string-credentials`: match `let $X: String = $CRED` where
       `$CRED` contains `password|secret|api_key|token|credential` (case-insensitive)
       and `$X` is not `SecretString` type; severity ERROR; message "Use
       `secrecy::SecretString` for credential values, never `String`."
     - Rule `prism-no-log-secret`: match `tracing::info!(...)` or `log::info!(...)`
       patterns where the format string contains `{$X}` and `$X` is a variable
       plausibly holding credentials; severity WARN; message "Do not log potential
       credential values."
   - `.semgrep/unsafe-patterns.yml`:
     - Rule `prism-unsafe-block`: match `unsafe { ... }` blocks outside of
       `#[allow(unsafe_code)]` attributed modules; severity WARN; message "Unsafe
       blocks require explicit `#[allow(unsafe_code)]` at module level with justification."
     - Rule `prism-unwrap-in-library`: match `.unwrap()` in crates that are not
       `prism-bin`; severity WARN; message "Use `?` or `expect()` with a descriptive
       message instead of `.unwrap()` in library crates."

9. Create `scripts/dev-setup.sh`:
   - Shebang: `#!/usr/bin/env bash`, `set -euo pipefail`
   - Check for `rustup` and `cargo` in PATH; exit with instructions if absent
   - Run `rustup update stable` and `rustup component add rustfmt clippy rust-src`
   - Install cargo tools (each wrapped in `cargo install --locked <tool>` with an
     existence check to be idempotent):
     - `cargo-deny`
     - `cargo-audit`
     - `cargo-semver-checks`
     - `cargo-mutants`
     - `cargo-fuzz`
     - `cargo-llvm-cov`
     - `just`
     - `lefthook`
   - Install Kani: `cargo install --locked kani-verifier && cargo kani setup`
   - Run `lefthook install` to activate git hooks in the current repo
   - Print success: "Development toolchain ready. Run 'just check' to verify."

10. Add the workspace `[features]` block to the root `Cargo.toml`:
    ```toml
    [features]
    # DTU (Deterministic Test Utility) sensor stubs — test infrastructure only.
    # Gate all prism-dtu code behind this feature so it is never compiled into
    # production binaries. Activated by `just integration-test` and `just dtu-start`.
    # NOTE: `just integration-test` and `just dtu-start` will fail with
    # "feature `dtu` not found" until S-6.06 (prism-dtu crate) is implemented.
    # This is expected and intentional — the Justfile targets exist as forward
    # declarations so that S-6.06 has a stable invocation target.
    dtu = []
    ```
    This is a workspace-level feature. Each crate that participates in DTU builds must
    declare `dtu = []` in its own `Cargo.toml` `[features]` section and gate DTU code
    with `#[cfg(feature = "dtu")]`.

## Acceptance Criteria

**AC-1:** Given a developer runs `just check` in a clean workspace, Then the command
runs the 6 PR gate steps (fmt check, clippy, test, deny, audit, semver-checks) in the
same order as `.github/workflows/ci.yml` and exits 0 on a clean codebase.

**AC-2:** Given a developer modifies a `.rs` file and runs `git commit`, When
`lefthook install` has been run previously, Then the pre-commit hook auto-runs
`cargo fmt` on the changed files and re-stages them, then runs `cargo clippy`, and
blocks the commit if clippy produces warnings.

**AC-3:** Given `scripts/dev-setup.sh` is run on a fresh macOS or Linux system with
Rust installed, When the script completes, Then all 9 cargo tool extensions (cargo-deny,
cargo-audit, cargo-semver-checks, cargo-mutants, cargo-fuzz, cargo-llvm-cov, just,
lefthook, kani-verifier) are available on the PATH.

**AC-4:** Given `scripts/dev-setup.sh` is run a second time on a system where all tools
are already installed, When the script completes, Then no errors occur (the script is
idempotent — it does not reinstall already-present tools).

**AC-5:** Given `cargo deny check` is run on the workspace, When `deny.toml` is present,
Then all crate licenses are checked against the allowlist and any crate with a
disallowed license causes the check to fail.

**AC-6:** Given a developer writes `let api_key: String = get_secret()`, When the
Semgrep SAST rule runs via `semgrep --config .semgrep/`, Then the rule
`prism-no-string-credentials` flags this as an ERROR.

## Architecture Compliance Rules

- `just check` MUST reproduce the CI gate exactly — if CI adds a step, `just check`
  must be updated in the same PR
- `clippy.toml` sets complexity thresholds but DOES NOT set lint levels — lint levels
  go in `[workspace.lints.clippy]` in root `Cargo.toml` so they apply to all crates
- `deny.toml` MUST use `wildcards = "deny"` — wildcard version requirements mask
  unintended dependency upgrades
- `lefthook.yml` MUST use `stage_fixed: true` for the fmt command — without it, the
  commit will include un-formatted code even though fmt ran
- Semgrep rules are WARN-level gates in local dev; they are informational and do not
  block `just check` (add semgrep to `just check` only after rules are validated)
- `rust-toolchain.toml` MUST pin `channel = "stable"` — never `nightly` for the
  primary toolchain (nightly is only used for `cargo udeps`)
