# S-3.4.03 — Migrate prism-dtu-crowdstrike tests to prism-dtu-harness

## Narrative

As a Prism integration-test maintainer, I want the `prism-dtu-crowdstrike` test suite
rewritten to use `prism-dtu-harness`, so that the larger CrowdStrike test matrix
(13 ACs, 2 integration tests, fidelity validator, 2 TD tests, edge cases) benefits from
multi-tenant isolation verification.

## Objective

Rewrite all `prism-dtu-crowdstrike` tests to use `HarnessBuilder`. The CrowdStrike
clone has the largest test matrix of any single-sensor DTU. All 13 original ACs, both
integration tests, and the edge case tests must pass post-migration. Add harness
isolation ACs. Both isolation modes exercised.

## Tasks

1. Catalog all existing test functions in `crates/prism-dtu-crowdstrike/tests/` (13 ACs, 2 integration tests, edge_cases, fidelity test, 2 TD tests).

2. Rewrite `tests/harness_tests.rs`:
   - All 13 ACs use harness-hosted clone (logical mode)
   - Both integration tests rewritten with harness
   - Edge case tests preserved and passing

3. Add 2-org logical isolation test (CrowdStrike): pairwise-disjoint device sets.

4. Add network isolation test: cross-org credential mismatch → HTTP 401.

5. Update fidelity validator to use `harness.endpoints()` for base URL.

6. Add `prism-dtu-harness` to `prism-dtu-crowdstrike/Cargo.toml` dev-deps.
