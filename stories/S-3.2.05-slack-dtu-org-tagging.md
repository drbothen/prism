# S-3.2.05 — prism-dtu-slack: Shared-mode OrgId ingress tagging

## Narrative

As a Prism MSSP operator, I want the Slack DTU clone to embed the `OrgId` UUID string
in every outgoing Slack payload body so that notifications from different client
organizations can be forensically attributed to their source without exposing org
identity in HTTP routing metadata (URLs, headers) visible to third-party Slack users.

## Objective

Update `prism-dtu-slack` to operate as `shared_infra = true` (metadata only), embed
`org_id` in captured payload bodies per ADR-007 §2.6 Step 3, and verify that mode is
immutably set at startup. The state struct itself is NOT re-keyed — shared-mode stores
remain globally shared per ADR-008 §1.2 and BC-3.2.004 invariant 2. Add `DtuMode::Shared`
metadata to sensor spec registration. Validate `mode = "shared"` enforcement at config
parse time per BC-3.2.005.

**Note on read-only enrichment DTUs (D-049):** NVD and ThreatIntel enrichment DTUs have
no per-org mutable state (their stores are immutable fixture registries). No state
re-keying is required for them. `OrgId` is accepted optionally at the route handler
level for audit attribution only, handled upstream of the adapter — not in the adapter's
state layer. This satisfies D-049 without requiring separate stories.

## Tasks

1. **Add `shared_infra` metadata to `SlackClone`** (`src/lib.rs` or config module):
   ```rust
   pub const DTU_MODE: DtuMode = DtuMode::Shared;
   ```
   Ensure `StubConfig` (from `prism-dtu-common`) is extended with `mode: DtuMode`
   where not already present.

2. **Inject `OrgId` in `capture_payload`** (`src/state.rs` line 153):
   Update the `capture_payload` method signature to accept `org_id: OrgId`:
   ```rust
   pub fn capture_payload(&self, org_id: OrgId, payload: Value) {
       let tagged = json!({
           "org_id": org_id.to_string(),  // UUID form, AI-opaque
           "payload": payload,
       });
       let mut received = self.received_payloads.lock().expect("poisoned");
       received.push(tagged);
   }
   ```
   The `OrgId` UUID string form is used (not the slug) per BC-3.2.004 invariant 1.

3. **Verify `OrgId` is absent from HTTP routing fields**:
   Inspect all route handler code paths to confirm no route uses `org_id` in:
   - URL path segments
   - URL query parameters
   - `X-` HTTP headers forwarded to Slack upstream API
   Add a unit test that inspects the HTTP request shape in captured payloads.

4. **Validate `mode = "shared"` at config parse** (in `prism-dtu-common` `StubConfig`
   or in `prism-config` startup pipeline):
   - `DtuMode` is `#[derive(Debug, Clone, Copy, PartialEq, Eq)]` enum.
   - Serde deserialization rejects values other than `"shared"` / `"client"`.
   - Security Telemetry DTU types (claroty, armis, crowdstrike, cyberint) reject
     `mode = "shared"` with a startup error.

5. **Validate `reload_config` does not apply mode changes**:
   If `reload_config` is exercised in tests, confirm that a mode edit in the TOML
   does not change the running `DtuMode` (BC-3.2.005 invariant 4).

6. **Add `tests/org_tagging.rs`**:
   - `test_org_id_in_payload_body` — send payload for org_A; assert captured payload
     JSON contains `"org_id": "<uuid-A>"`.
   - `test_org_id_not_in_http_url` — send payload; assert HTTP request URL has no
     org_id or org_slug component.
   - `test_concurrent_sends_distinguished` — send for org_A and org_B concurrently;
     assert both payloads contain their respective `org_id` UUIDs.
   - `test_mode_immutable_after_startup` — load config with `mode = "shared"`;
     attempt to change via in-process API; confirm DtuMode unchanged.

7. **Gate**: `cargo test -p prism-dtu-slack` green.

## Acceptance Criteria

**AC-001** — OrgId in payload body (traces to BC-3.2.004 postcondition 1):
Given org_A sends an incident notification to the shared Slack DTU,
When the captured payload is inspected,
Then the JSON contains `"org_id": "<uuid-A>"` in the payload body; the HTTP request
URL contains no org_id or org_slug component.

**AC-002** — OrgId absent from HTTP routing fields (traces to BC-3.2.004 postcondition 2):
Given any outgoing Slack payload,
When the HTTP request shape is inspected,
Then the URL path, URL query parameters, and `X-` headers contain no OrgId or OrgSlug string.

**AC-003** — Concurrent sends produce independent payloads (traces to BC-3.2.004 postcondition 4):
Given org_A and org_B both send payloads concurrently to the shared Slack DTU,
When both payloads are captured and inspected,
Then each contains its sender's OrgId UUID and they are distinct.

**AC-004** — Mode metadata absent from query results (traces to BC-3.2.004 postcondition 5):
Given analyst queries sensor data via this shared DTU,
When the result rows are inspected,
Then no `"mode"`, `"shared"`, or `"org_routing"` fields appear in OCSF-normalized event records.

**AC-005** — DtuMode::Shared set at startup (traces to BC-3.2.005 postcondition 1):
Given config TOML has `mode = "shared"` for the Slack DTU,
When the process starts,
Then the registered DTU mode is `DtuMode::Shared` for the lifetime of the process.

**AC-006** — Invalid mode string rejected at startup (traces to BC-3.2.005 postcondition 3):
Given config TOML has `mode = "Hybrid"` for any DTU,
When the process attempts to start,
Then serde deserialization fails with a human-readable error identifying the offending key.

**AC-007** — reload_config does not change mode (traces to BC-3.2.005 invariant 4):
Given the process is running with `DtuMode::Shared`,
When the TOML is edited to `mode = "client"` and `reload_config` is called,
Then the running `DtuMode` remains `Shared` (no restart applied).

## Architecture Compliance Rules

- `OrgId` MUST appear in the payload JSON body — never in URL path, query parameter,
  or forwarded HTTP header. This is BC-3.2.004 invariant 1 and is validated by
  `test_org_id_not_in_http_url`.
- `OrgId` UUID string form (not OrgSlug) MUST be used in payload bodies (AI-opacity
  principle per `project_ai_opaque_credentials.md`).
- The shared Slack state store (`received_payloads: Vec<Value>`) MUST NOT be re-keyed
  by OrgId — it remains globally shared per ADR-008 §1.2. OrgId in the payload is
  metadata/attribution only.
- **Forbidden dependencies:** `prism-dtu-slack` MUST NOT depend on `prism-sensors`,
  `prism-query`, `prism-operations`, `prism-mcp`, or `prism-spec-engine`.
- `DtuMode` enum MUST be `#[derive(Debug, Clone, Copy, PartialEq, Eq)]` — no interior
  mutability; no setter methods post-startup.
