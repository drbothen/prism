# S-1.14-REDO — prism-spec-engine: Infusion Engine REDO

## Narrative

As the Prism query engine, I want the infusion enrichment framework fully implemented —
loading `.infusion.toml` fixtures via `InfusionLoader`, maintaining a live `InfusionRegistry`
with arc-swap hot reload, registering enrichment sources as DataFusion UDFs in a
`SessionContext`, executing `| enrich type=...` pipe stages per query, deduplicating
enrichment calls via a per-query cache, and persisting enrichment results in the RocksDB
`infusion_cache` column family — so that analysts can enrich query results with GeoIP,
threat intelligence, and asset inventory context using the `| enrich` pipe stage.

## Objective

Implement all `unimplemented!()` stubs in the infusion module:
- `InfusionLoader::parse`, `load_all`, `validate_credentials` (loader.rs:42/50/57/66)
- `InfusionLruCache::get`, `insert` (cache.rs:109/120)
- `MmdbSource::load`, `enrich_single`, `enrich_batch` (sources/mmdb.rs:23/29/37)
- CsvSource and JsonLookupSource equivalents
- `plugin_bridge::enrich_via_plugin` (plugin_bridge.rs:26/37) — delegates to WASM runtime (S-1.15)
- DataFusion UDF registration for `enrich(source, field)` expression (in prism-query, called from `InfusionRegistry::udf_descriptors()`)
- Pipe stage `| enrich <source>` compilation to UDF invocation

Architect REDO decision (ADR-022 §D): wasmtime 44 is already in workspace deps. Scaffolding
is real. Implementation cost < retirement cost. REDO at Wave 5.

No `unimplemented!()` or `todo!()` may remain in any infusion stub before this story
merges, EXCEPT `plugin_bridge::enrich_via_plugin` which delegates to S-1.15 (WASM runtime)
and may remain as a well-annotated `todo!("S-1.15 — WASM runtime required")` if S-1.15
is not yet merged at dispatch time. Per POL-12 (exception documented).

## REDO Scope: What Stays vs. What Is Replaced

This is a REDO of S-1.14, not a greenfield implementation. Review `S-1.14-infusion-specs.md`
before implementing.

### What Stays As-Is (preserved from S-1.14)
- `InfusionSpec`, `InfusionField`, `InfusionRegistry` struct definitions (types are correct)
- `InfusionSource` trait definition: `enrich_single` + `enrich_batch` signatures
- `InfusionUdfDescriptor` type (export shape is correct)

- Test fixture files: `test.mmdb`, `geoip.infusion.toml`, `asset_inventory.csv`
- VP-048 Kani proof harness skeleton at `proofs/infusion_spec.rs` (blocked on wasmtime upstream)
- VP-049 proptest structure at `proofs/infusion_dedup.rs`
- `is_api_backed(udf_name)` method interface (signature stays; body may be stubbed)

### What Is Replaced/Implemented (the REDO work)
- `InfusionLoader::parse/load_all/validate_credentials` — implement TOML loading real logic
- `InfusionLruCache::get/insert` — implement LRU cache with TTL using `lru` crate
- `MmdbSource::load/enrich_single/enrich_batch` — implement via `maxminddb` crate
- `CsvSource` — implement HashMap-backed lookup
- `JsonLookupSource` — implement JSON file-backed lookup
- RocksDB `infusion_cache` CF integration: `QueryScopedInfusionCache` per-query dedup, Tier 3 read/write
- DataFusion UDF registration: `InfusionRegistry::udf_descriptors()` called by prism-query to register UDFs in `SessionContext`
- Pipe stage `| enrich <source>` → compile to UDF invocation in DataFusion plan
- `plugin_bridge::enrich_via_plugin` — delegate to S-1.15; annotated `todo!()` if S-1.15 unavailable

## Tasks

1. Implement `InfusionLoader::parse` at `loader.rs:42`:
   - Read `*.infusion.toml` from `{config_dir}/infusions/` directory.
   - Deserialize using `toml` crate into `InfusionSpec`.
   - Validate: `infusion_id` present, at least one `[[infusion.fields]]` entry, `source.type`
     is one of `maxmind_mmdb | csv | json_lookup | plugin`.
   - Validate credential references use reference-based model (no inline values per AD-017).
   - Validate `pipe_stage.adds_columns` matches `[[infusion.fields]]` names.
   - On any validation error: return `Err(SpecEngineError::InfusionValidation(...))`.

2. Implement `InfusionLoader::load_all` at `loader.rs:50`:
   - Call `parse` for each `.infusion.toml` found in the directory.
   - For each valid spec: construct the appropriate `InfusionSource` impl:
     - `source.type = "maxmind_mmdb"` → `MmdbSource::load(path)`
     - `source.type = "csv"` → `CsvSource::load(path, key_column)`
     - `source.type = "json_lookup"` → `JsonLookupSource::load(path)`
     - `source.type = "plugin"` → `PluginInfusionSource` (delegates to S-1.15 WASM)
   - Return `InfusionRegistry` with all loaded specs + sources.

3. Implement `InfusionLoader::validate_credentials` at `loader.rs:57/66`:
   - For each `InfusionSpec` with `[infusion.credentials]` entries, verify credential refs
     resolve (exist in `CredentialStore`) without loading actual values.
   - If resolution fails: return `Err` with the ref name but NOT the credential value.
   - Per BC-2.19.005: credential values must never appear in errors or logs.

4. Implement `MmdbSource::load/enrich_single/enrich_batch` at `sources/mmdb.rs:23/29/37`:
   ```rust
   pub struct MmdbSource {
       reader: maxminddb::Reader<Vec<u8>>,
       field_names: Vec<String>,  // declared output fields from spec
   }
   impl MmdbSource {
       pub fn load(path: &Path, field_names: Vec<String>) -> Result<Self, SpecEngineError> {
           let reader = maxminddb::Reader::open_readfile(path)?;
           Ok(Self { reader, field_names })
       }
       pub fn enrich_single(&self, ip: &str, _input_type: &str) -> Option<serde_json::Value> {
           let addr: std::net::IpAddr = ip.parse().ok()?;
           let record: serde_json::Value = self.reader.lookup(addr).ok()?;
           Some(record)
       }
       pub fn enrich_batch(&self, ips: &[String], input_type: &str) -> Vec<Option<serde_json::Value>> {
           ips.iter().map(|ip| self.enrich_single(ip, input_type)).collect()
       }
   }
   ```

5. Implement `CsvSource` and `JsonLookupSource` per S-1.14 Task 3 design. These are
   straightforward HashMap-backed loaders — the S-1.14 spec defines the shape; implement it.

6. Implement `InfusionLruCache::get/insert` at `cache.rs:109/120`:
   ```rust
   pub struct InfusionLruCache {
       inner: tokio::sync::Mutex<lru::LruCache<String, (serde_json::Value, std::time::Instant)>>,
       ttl: std::time::Duration,
   }
   impl InfusionLruCache {
       pub async fn get(&self, key: &str) -> Option<serde_json::Value> {
           let mut cache = self.inner.lock().await;
           if let Some((v, inserted)) = cache.get(key) {
               if inserted.elapsed() < self.ttl { return Some(v.clone()); }
               cache.pop(key);
           }
           None
       }
       pub async fn insert(&self, key: String, value: serde_json::Value) {
           self.inner.lock().await.put(key, (value, std::time::Instant::now()));
       }
   }
   ```

7. Implement `QueryScopedInfusionCache` (Tier 1 per-query dedup):
   ```rust
   pub struct QueryScopedInfusionCache {
       map: HashMap<String, Option<serde_json::Value>>,  // key = "{infusion_id}:{input_value}"
   }
   ```
   Allocated fresh for each `QueryEngine::execute()` call. Dropped when the query
   completes. Passed into the DataFusion UDF execution context via `SessionState` extension.

8. Wire Tier 3 RocksDB cache:
   - Key format: `"{infusion_id}:{sha256(input_value)}"` (use `sha2` crate).
   - Value: bincode 2.x encoding of `(serde_json::Value, expiry_unix_secs: u64)`.
   - On read: if `expiry_unix_secs < now()`, discard (lazy eviction).
   - Write TTL from `infusion_spec.cache_ttl_secs` field (default 3600s).
   - Use `CacheBackend` trait injected at startup (prism-spec-engine does NOT import
     prism-storage directly; trait injection per S-1.14 design).
   - Hot path: Tier 1 (query-scoped HashMap) → Tier 2 (LRU) → Tier 3 (RocksDB) → live source.

9. Wire DataFusion UDF registration:
   - `InfusionRegistry::udf_descriptors()` → `Vec<InfusionUdfDescriptor>`.
   - In `S-3.02-FOLLOWUP-RUNTIME` (prism-query), call `register_infusion_udfs(ctx, descriptors)`:
     ```rust
     for desc in registry.udf_descriptors() {
         let udf = ScalarUDF::from(InfusionScalarUdf::new(desc, cache.clone()));
         ctx.register_udf(udf);
     }
     ```
   - The `InfusionScalarUdf` struct implements DataFusion's `ScalarUDFImpl`. It calls
     `Tier1Cache → Tier2 → Tier3 → live source` on each invocation.
   - NOTE: This wiring step touches `prism-query` (which depends on `prism-spec-engine`).
     Coordinate with S-3.02-FOLLOWUP-RUNTIME implementer — the UDF registration call goes
     into `QueryEngine::new` or a post-construction `register_infusion_udfs` call.

10. Implement pipe stage `| enrich <source>` compilation:
    - After the upstream query produces a `RecordBatch`, call
      `InfusionRegistry::enrich_descriptor(source_name)` to get the descriptor.
    - Extract unique values of the ON-field from the batch.
    - Call `InfusionSource::enrich_batch(unique_values)` (dedup via Tier 1 cache).
    - Join new columns back to the `RecordBatch`.
    - Unknown source name → `E-INFUSE-001`.

11. Implement `plugin_bridge::enrich_via_plugin` at `plugin_bridge.rs:26/37`:
    - If S-1.15 (WASM runtime) is merged: call `PluginRuntime::enrich_single/batch`.
    - If S-1.15 is NOT yet merged at dispatch time:
      ```rust
      // TODO(S-1.15): WASM plugin runtime required for plugin-type infusions.
      // Local lookup types (mmdb, csv, json_lookup) work independently.
      Err(SpecEngineError::PluginRuntimeNotAvailable)
      ```
      This is the ONLY `todo!()` exemption in this story. Document it in the PR.

12. Run VP-048 and VP-049:
    - VP-048 (Kani): `InfusionRegistry::load_spec` with N distinct fields → N descriptors.
      Upstream blocker (wasmtime Linker enumeration API) may keep this at `harness-only`.
      Status: check current VP-048 status in VP-INDEX before dispatching.
    - VP-049 (proptest): run to verify per-query dedup — N values with K distinct → `enrich_single` called K times.

## Acceptance Criteria

**AC-1:** Given `geoip.infusion.toml` with `type = "local_lookup"` and `source.type = "maxmind_mmdb"`,
When `InfusionLoader::load_all` runs, Then `geoip_country`, `geoip_city`, `geoip_asn`, and
`geoip_is_tor` are exported as `InfusionUdfDescriptor` entries from `InfusionRegistry`.
(traces to BC-2.19.001 postcondition — each field registers exactly one UDF descriptor)

**AC-2:** Given a query `SELECT geoip_country(device_ip) FROM EVENTS` executed against a result
set with 500 events having 30 unique IPs, When the query executes with the per-query dedup cache
active, Then `MmdbSource::enrich_single` is called exactly 30 times (not 500 times).
(traces to BC-2.19.002 postcondition — unique input values only)

**AC-3:** Given a pipe stage `| enrich geoip ON device_ip`, When the query executes, Then the
output schema includes columns `geoip_country`, `geoip_city`, `geoip_asn`, `geoip_is_tor`
added to the upstream result schema.
(traces to BC-2.19.001 postcondition — enrich pipe stage produces declared output columns)

**AC-4:** Given an infusion `threat_intel` with `type = "plugin"`, When a detection rule
references `threat_score(device_ip)` in its `filter` expression, Then `is_api_backed("threat_score")`
returns `true` and the detection rule loader rejects with `E-RULE-012`.
(traces to BC-2.19.003 postcondition — API-backed UDFs rejected in detection rules)

**AC-5:** Given a valid infusion spec loaded, When the `.infusion.toml` file is modified and
the file watcher fires (via S-1.12-FOLLOWUP), Then the `InfusionRegistry` is updated atomically
and in-flight queries using the old registry complete without error.
(traces to BC-2.19.004 postcondition — hot reload atomicity, CI-002 invariant)

**AC-6:** Given an infusion spec with `[infusion.credentials]` entries, When the loader returns
an error, Then no credential values appear in any log output or error message (only ref names).
(traces to BC-2.19.005 postcondition — credential redaction)

**AC-7:** Given an infusion result fetched from a live source, When it is stored in RocksDB
`infusion_cache` CF, Then a subsequent query (after Tier 1 and Tier 2 LRU have expired) reads
from RocksDB without calling the live source again.
(traces to BC-2.19.002 postcondition — three-tier cache reduces live source calls)

**AC-8:** VP-049 proptest passes: for any input sequence of N values with K distinct (K <= N),
`enrich_single` is called exactly K times.
(traces to BC-2.19.002 postcondition)

**AC-9:** No `unimplemented!()` or `todo!()` may remain in `loader.rs`, `cache.rs`,
`sources/mmdb.rs`, `sources/csv.rs`, `sources/json_lookup.rs` before merge.
`plugin_bridge.rs` may contain a single annotated `todo!("S-1.15 — WASM runtime required")`
if S-1.15 is not yet merged. Per POL-12.
