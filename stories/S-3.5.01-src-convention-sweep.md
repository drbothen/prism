# S-3.5.01 — Workspace src/ convention sweep

## Narrative

As a Prism contributor, I want a canonical crate layout enforced by a pre-commit hook
and CI gate so that every workspace crate — current and future — conforms to the
`src/lib.rs` convention, fixture data lives in `crates/<name>/fixtures/` (not
`tests/fixtures/`), and violations are caught before merge rather than discovered
during code review.

## Objective

Implement the ADR-012 enforcement infrastructure:
1. Migrate `crates/prism-spec-engine/tests/fixtures/` → `crates/prism-spec-engine/fixtures/` (the one active deviation).
2. Create `scripts/check-crate-layout.sh` — a read-only validator that exits 0 (all pass) or non-zero (violations listed).
3. Add `just check-layout` Justfile target.
4. Add `lefthook.yml` `layout` pre-commit command.
5. Append `just check-layout` to `just check` (CI step).
6. Write `docs/CRATE-LAYOUT.md` — canonical shape, rules, FAQ, 22-crate conformance table.

Per D-060, `BC-3.7.001` subsystem is SS-01 primary with cross-cutting note (all 6 subsystems affected).

## Tasks

1. **Migrate `prism-spec-engine` fixtures** (ADR-012 §3 Step 1):
   ```bash
   mv crates/prism-spec-engine/tests/fixtures/ crates/prism-spec-engine/fixtures/
   ```
   Update all path references in `crates/prism-spec-engine/tests/` from
   `"tests/fixtures/<name>"` to `"fixtures/<name>"` (use `env!("CARGO_MANIFEST_DIR")`
   prefix for portability: `concat!(env!("CARGO_MANIFEST_DIR"), "/fixtures/<name>")`).
   Gate: `cargo test -p prism-spec-engine` green.

2. **Create `scripts/check-crate-layout.sh`** (ADR-012 §2.4):
   The script scans `crates/` for workspace crates and checks each against:
   - Rule 1: No `.rs` files at crate root (except `build.rs`).
   - Rule 2: `src/lib.rs` OR `src/main.rs` exists.
   - Rule 3: No fixture files under `tests/fixtures/` (should be in `fixtures/`).
   - Rule 4: `inline #[cfg(test)]` in `src/` is permitted — not checked.
   - Rule 5: `src/tests/` submodule is permitted — not checked.
   - Exception: `prism-ocsf` — no `tests/` directory is conformant (tests/ is optional).
   - Exception: `build.rs` at crate root is explicitly excluded from Rule 1.
   
   Exit codes:
   - 0: all crates conform; no output (or optional summary line).
   - Non-zero: per-crate violation lines in format:
     `"crates/<name>: <rule description>"`
   
   The script is READ-ONLY — it does not modify any file.
   
   Add `--markdown` flag for `just layout-report` output to `docs/CRATE-LAYOUT.md`.

3. **Add `just check-layout` target** to `Justfile`:
   ```just
   # Validate canonical src/ layout for all workspace crates.
   check-layout:
       @scripts/check-crate-layout.sh
   ```

4. **Add `layout` command to `lefthook.yml`** (ADR-012 §2.5):
   ```yaml
   layout:
     glob: "crates/**"
     run: scripts/check-crate-layout.sh
   ```
   Runs only when files under `crates/` are staged. Runtime must be under 1 second
   for the current 22-crate workspace.

5. **Append `just check-layout` to `just check`** (ADR-012 §2.6):
   Add as the final step in the existing `check` target so CI runs it on every push.

6. **Write `docs/CRATE-LAYOUT.md`** (ADR-012 §2.3):
   - Canonical shape directory tree (Section 2.1 of ADR-012).
   - Rules table (rules 1-6).
   - FAQ: "Where do fixture files go?", "Where do integration tests go?",
     "Inline unit tests or `src/tests/`?", "Is `tests/` required?" (answer: no — `prism-ocsf` has none).
   - 22-crate conformance table (auto-populated by `just check-layout --markdown`).
   - `build.rs` exception documented explicitly.
   - `prism-ocsf` no-`tests/` exception documented explicitly.

7. **Write tests for the script**:
   Create `tests/check_crate_layout_test.rs` (or a shell test via `bats` if preferred):
   - `test_all_existing_crates_pass` — run `scripts/check-crate-layout.sh` against the
     workspace; assert exit 0 and no violation lines.
   - `test_synthetic_bad_crate_fails` — create a temp directory with `lib.rs` at root
     (no `src/`); run script; assert non-zero exit and violation line naming the crate.
   - `test_tests_fixtures_triggers_violation` — temp dir with `src/lib.rs` + `tests/fixtures/data.json`;
     assert violation: "fixtures should be in fixtures/, not tests/fixtures/".
   - `test_prism_ocsf_no_tests_passes` — temp dir mimicking prism-ocsf (no `tests/`); assert exit 0.
   - `test_build_rs_at_root_permitted` — temp dir with `src/lib.rs` + `build.rs`; assert exit 0.
   - `test_script_is_readonly` — run script; compare `git status` before and after; assert no file changes.

8. **Gate**: `just check-layout` exits 0 on current workspace; `just check` includes it.

## Acceptance Criteria

**AC-001** — Conformant crate passes (traces to BC-3.7.001 postcondition 1):
Given all 22 existing workspace crates in their current state after the fixture migration,
When `just check-layout` is run,
Then it exits 0 and produces no violation lines for any existing crate.

**AC-002** — Non-conformant crate fails with violation line (traces to BC-3.7.001 postcondition 2):
Given a synthetic crate at `crates/test-bad-crate/` with `Cargo.toml` + `lib.rs` at root (no `src/`),
When `just check-layout` is run,
Then it exits non-zero and prints a violation line identifying `crates/test-bad-crate` and the missing `src/lib.rs` rule.

**AC-003** — Violation output names crate and rule (traces to BC-3.7.001 postcondition 3):
Given a non-conformant crate,
When `just check-layout` runs,
Then the output line format is `"crates/<name>: <rule description>"` — both the crate path and violated rule are present.

**AC-004** — Pre-commit hook blocks bad crate before commit (traces to BC-3.7.001 postcondition 4):
Given a contributor stages `crates/bad-crate/lib.rs` (no `src/`),
When `git commit` is attempted,
Then the `lefthook` `layout` hook runs, prints the violation, and exits non-zero — blocking the commit.

**AC-005** — `prism-ocsf` without `tests/` is conformant (traces to BC-3.7.001 postcondition 5, edge case EC-003):
Given `prism-ocsf` has `Cargo.toml` and `src/lib.rs` but no `tests/` directory,
When `just check-layout` runs,
Then no violation line appears for `prism-ocsf` — `tests/` is optional.

**AC-006** — `prism-spec-engine` fixture migration complete (traces to BC-3.7.001 postcondition 6):
Given `crates/prism-spec-engine/tests/fixtures/` previously existed,
When this story is implemented,
Then `crates/prism-spec-engine/fixtures/` exists; `tests/fixtures/` does not; `cargo test -p prism-spec-engine` is green.

**AC-007** — `build.rs` at crate root is not flagged (traces to BC-3.7.001 postcondition 7, edge case EC-004):
Given a crate with `Cargo.toml`, `src/lib.rs`, and `build.rs` at the root,
When `just check-layout` runs,
Then no violation — `build.rs` is the Cargo-mandated build script location and is explicitly excluded.

**AC-008** — Script is read-only (traces to BC-3.7.001 invariant 2):
Given `just check-layout` is run against the full workspace,
When the run completes,
Then no file in the workspace has been created, modified, or deleted by the script.

**AC-009** — CI rejects non-conformant PR (traces to BC-3.7.001 postcondition 4):
Given `just check-layout` is appended to `just check`,
When a PR introducing a non-conformant crate triggers CI,
Then the `check` step exits non-zero and the PR is blocked from merge.

## Architecture Compliance Rules

- `scripts/check-crate-layout.sh` MUST be read-only: zero file creates, modifies, or
  deletes. VP-136 validates this.
- The script MUST exit 0 for all 22 existing crates after the prism-spec-engine fixture
  migration (VP-134).
- The script MUST exit non-zero for a synthetic non-conformant fixture (VP-135).
- `prism-ocsf` with no `tests/` directory MUST NOT be flagged as a violation.
- `build.rs` at any crate root MUST NOT be flagged as a violation — explicit exclusion
  from the "no loose .rs at crate root" rule.
- The `layout` pre-commit hook glob `"crates/**"` runs the full workspace scan once
  per commit — NOT once per staged file. Runtime MUST be under 1 second.
- `just check-layout` is the FINAL step in `just check` — it must not break the
  existing CI step order.
- `docs/CRATE-LAYOUT.md` MUST document the `prism-ocsf` no-`tests/` exception
  explicitly to prevent false violation reports.

## Edge Cases

| ID | Description | Expected Behavior |
|----|-------------|-------------------|
| EC-001 | New crate created with `lib.rs` at root (no `src/`) | Script exits non-zero; violation line names the crate |
| EC-002 | New crate has `src/lib.rs` and `tests/fixtures/data.json` | Script exits non-zero; violation: "fixtures should be in fixtures/" |
| EC-003 | `prism-ocsf` has no `tests/` directory | Script exits 0; no violation (tests/ is optional) |
| EC-004 | `build.rs` at crate root alongside `src/lib.rs` | Script exits 0; build.rs explicitly excluded |
| EC-005 | Workspace grows to 30 crates; no conformance drift | Script still exits 0; runtime stays under 1s |
| EC-006 | `src/tests/mod.rs` in `prism-core` (existing pattern) | Not flagged; `src/tests/` is a permitted dedicated test submodule |
| EC-007 | `.rs` file at crate root with name other than `build.rs` (e.g., `helpers.rs`) | Script exits non-zero; violation: "loose .rs file at crate root: helpers.rs" |
