# S-6.03 — prism-bin: Installation and Distribution

## Narrative

As an MSSP analyst or operator, I want to install Prism with a single command, register
it with Claude Code in one step, and optionally bootstrap with a pre-built query pack,
so that I can go from zero to a working Prism+Claude Code setup in under 5 minutes.

## Objective

Implement distribution and installation tooling: `cargo install` support, the
`prism register` command for Claude Code MCP config, the
`prism install-secops-factory` command for query pack installation, a Homebrew formula
shell, and a README quickstart that guides through the full installation flow.

## Tasks

1. Configure `crates/prism-bin/Cargo.toml` for `cargo install` support:
   - Ensure `[[bin]] name = "prism"` is declared
   - Set `description`, `homepage`, `repository`, `license` in `[package]`
   - Pin `rust-version` to the MSRV for this project
   - Verify `cargo install prism` resolves all dependencies without conflict

2. Implement `prism register` in `crates/prism-bin/src/commands/register.rs`:
   - Detect the Claude Code MCP config file path:
     1. `$CLAUDE_CONFIG_DIR/mcp_servers.json` (if env var set)
     2. `~/.config/claude/mcp_servers.json` (Linux/macOS default)
   - If the file does not exist: create it with an empty `{ "mcpServers": {} }` object
   - Read existing config, add or update a `"prism"` entry:
     ```json
     "prism": {
       "command": "prism",
       "args": ["serve"],
       "env": {}
     }
     ```
   - Write updated config back atomically (write to `.tmp` then rename)
   - Print: "Registered Prism in Claude Code MCP config at <path>"
   - If a `"prism"` entry already exists: print a warning and overwrite (do not fail)

3. Implement `prism install-secops-factory` in
   `crates/prism-bin/src/commands/install_pack.rs`:
   - Determine install directory: `$XDG_CONFIG_HOME/prism/packs/` or
     `~/.config/prism/packs/` fallback
   - Accept optional `--source <URL_OR_PATH>` flag; default to a hardcoded public git
     archive URL for the secops-factory pack
   - Download the pack archive (`.tar.gz`) using `reqwest::blocking::get` or stream
   - Extract into the packs directory, overwriting if already present
   - Print: "Installed secops-factory pack to <path> — add 'pack_dirs = [\"<path>\"]'
     to your prism.toml to activate it"

4. Create Homebrew formula shell at `Formula/prism.rb`:
   - Standard Ruby formula structure with `desc`, `homepage`, `url`, `sha256`
   - `bottle` block placeholder (populated by release workflow)
   - `depends_on` for any system dependencies (none expected)
   - `install`: `system "cargo", "install", "--locked", "--root", prefix, "--path", "."`
   - `test`: `system "#{bin}/prism", "--version"`
   - Note in a comment that sha256 and url are populated by the release workflow

5. Write README quickstart in `README.md` (create if absent, or add section):
   - Section: "Quick Start"
   - Step 1: `cargo install prism` (with expected output)
   - Step 2: `prism init` (creates prism.toml, explain what to edit)
   - Step 3: Edit `prism.toml` — add at least one client and credential_ref
   - Step 4: `prism serve --dry-run` — validate config before first run
   - Step 5: `prism register` — wire into Claude Code
   - Step 6: Restart Claude Code — Prism MCP tools now appear in tool list
   - Step 7 (optional): `prism install-secops-factory` — add curated query pack
   - Each step includes the exact command to run and what to expect in output

## Acceptance Criteria

**AC-1:** Given `cargo install --path crates/prism-bin`, When run in the workspace root,
Then a working `prism` binary is installed to `~/.cargo/bin/prism` and `prism --version`
prints a version string.

**AC-2:** Given `prism register` run by a user without an existing Claude Code MCP config,
When executed, Then `~/.config/claude/mcp_servers.json` is created and contains a
`"prism"` entry with `"command": "prism"` and `"args": ["serve"]`.

**AC-3:** Given `prism register` run by a user who already has a Claude Code MCP config
with other server entries, When executed, Then the existing entries are preserved and a
`"prism"` entry is added without corrupting the file.

**AC-4:** Given `prism install-secops-factory`, When run, Then the pack is extracted into
`~/.config/prism/packs/secops-factory/` and a message is printed with the path and
activation instructions.

**AC-5:** Given the README quickstart, When followed exactly by a new user, Then they
reach a working Prism + Claude Code installation with at least one sensor configured.
(This is a manual verification criterion — verified by team walkthrough, not automated
test.)

## Architecture Compliance Rules

- `prism register` MUST write the MCP config file atomically (write to `.tmp` then
  rename) to avoid corrupting the config if the process is interrupted
- `prism register` MUST NOT require root or sudo — it writes only to user-owned paths
- The `install-secops-factory` command MUST validate the downloaded archive before
  extracting (check Content-Type or file magic) to prevent extracting corrupt data
- The Homebrew formula MUST use `--locked` when invoking `cargo install` to ensure
  reproducible builds from `Cargo.lock`
- The README quickstart commands MUST be copy-paste safe — no placeholder text that
  looks like a real command but isn't

## Dev Notes

- The secops-factory query pack URL should be configurable at compile time via a
  `PRISM_SECOPS_FACTORY_URL` environment variable at build time, with a sensible default.
  This allows forks and private deployments to point to their own pack repositories.
- The Homebrew formula is a shell at this stage — the actual sha256 and release URL
  are filled in by the release workflow (not this story). Include a prominent comment
  explaining this so the next engineer doesn't wonder why the formula has placeholder
  values.
- The atomic write for `mcp_servers.json` is important: Claude Code may be running
  while `prism register` executes, and a partial write would break Claude Code's MCP
  config loading.
- README quickstart should note that after `prism register`, Claude Code requires a
  restart to pick up the new MCP server. This is a Claude Code behavior, not a Prism
  limitation, but users will otherwise be confused when tools don't appear immediately.
