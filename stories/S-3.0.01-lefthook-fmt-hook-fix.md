# S-3.0.01 — lefthook: fix pre-commit fmt hook (cargo fmt --all --check)

## Narrative
- **As a** Prism contributor
- **I want** the lefthook pre-commit `fmt` hook to run `cargo fmt --all --check` (workspace-wide)
- **So that** every commit is blocked on formatting violations without requiring `LEFTHOOK=0` bypass workarounds

## Description

Closes tech debt item TD-W2-FIX-H-001. The `lefthook.yml` pre-commit `fmt` command
currently runs `cargo fmt --check {staged_files}`, passing per-file paths as positional
arguments to stable `cargo fmt`. Stable `cargo fmt` does not accept positional file path
arguments; the hook silently no-ops or errors depending on toolchain version. Every Wave 2
PR delivery required bypassing lefthook with `LEFTHOOK=0` after a manual
`cargo fmt --all --check`. This story replaces the broken invocation with the correct
workspace-wide form.

## Acceptance Criteria

**AC-1:** `lefthook.yml` pre-commit `fmt` command is changed from
`cargo fmt --check {staged_files}` to `cargo fmt --all --check`.
*(tooling fix — no BC; validated by diff review)*

**AC-2:** A workspace `.rs` file containing unformatted code (trailing whitespace, wrong
indentation, etc.) causes the pre-commit hook to return a non-zero exit code, blocking
the commit.
*(tooling fix — no BC; validated by manual smoke test)*

**AC-3:** A fully formatted workspace allows the pre-commit hook to pass with exit code 0.
*(tooling fix — no BC; validated by manual smoke test)*

**AC-4:** The `stage_fixed` setting in `lefthook.yml` is explicitly resolved: either
removed (since `--check` does not modify files) or the command is changed to
`cargo fmt --all` (auto-fix) so `stage_fixed: true` operates correctly. Decision is
documented in the PR description.
*(tooling fix — no BC)*
