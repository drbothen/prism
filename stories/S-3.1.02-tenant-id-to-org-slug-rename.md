# S-3.1.02 — workspace: rename TenantId → OrgSlug across all crates

## Narrative
- **As a** developer working across the Prism workspace
- **I want** the `TenantId` type to be renamed to `OrgSlug` everywhere in the codebase
- **So that** the type name correctly reflects its role (the analyst-visible display
  identifier, not the canonical key) and downstream stories can use `OrgSlug` without
  ambiguity

## Description

ADR-006 §2.1 (D-041) renames `TenantId` → `OrgSlug` across the workspace. The type
semantics are unchanged: the regex `^[a-zA-Z0-9_-]{1,64}$` is preserved (locked at 64
chars per D-041), the underlying `Arc<str>` representation is preserved, and all
validation logic is preserved. The rename is mechanical — `cargo rename` + grep sweep.
A one-wave deprecation alias `pub type TenantId = OrgSlug;` is added in
`prism-core/src/tenant.rs` to ease migration; it will be removed in Wave 4.

## Acceptance Criteria

**AC-1 (traces to BC-3.1.001 precondition 2):** `OrgSlug` is declared in
`crates/prism-core/src/tenant.rs` (or a new `org_slug.rs` module re-exported from it)
with the same regex constant `^[a-zA-Z0-9_-]{1,64}$` as the former `TenantId`.
The constant is renamed from `TENANT_ID_PATTERN` to `ORG_SLUG_PATTERN`.

**AC-2 (traces to BC-3.1.001 precondition 2):** `OrgSlug::try_new(s)` accepts strings
matching `ORG_SLUG_PATTERN` and rejects all others with the same error type as the
former `TenantId::try_new`. All existing `TenantId` validation unit tests pass
after name substitution.

**AC-3 (traces to BC-3.1.001 invariant 2):** A deprecation alias
`#[deprecated(since = "3.0.0", note = "use OrgSlug")] pub type TenantId = OrgSlug;`
is present in `prism-core/src/tenant.rs` for exactly one wave. All call sites in
`prism-credentials`, `prism-spec-engine`, `prism-sensors`, and `prism-audit` are
updated to use `OrgSlug` directly (no remaining non-deprecated `TenantId` usages in
non-test code).

**AC-4 (traces to BC-3.1.001 invariant 1):** `cargo test --workspace` passes with
zero compilation errors and zero test failures after the rename sweep. The workspace
treats any `TenantId` deprecation warning as a warning, not an error, during the
migration wave.
