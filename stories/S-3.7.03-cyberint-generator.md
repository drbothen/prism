# S-3.7.03 — Cyberint fixture generator

## Narrative

As a test harness consumer, I want a `generate(org_id, SensorType::Cyberint, archetype, opts)`
implementation backed by the four poller-express OpenAPI specs, so that integration tests
cover all Cyberint API surfaces (alert, ASM asset, CVE, IOC) across all 8 archetypes
without live API calls.

## Acceptance Criteria

### AC-001 — All 8 archetypes produce records for all 4 Cyberint API surfaces (traces to BC-3.4.002 postcondition — Cyberint row)

`generate(org, Cyberint, archetype, GenOpts::default())` produces a `FixtureSet` whose
records are tagged by surface type (`alert`, `asm_asset`, `cve`, `ioc`). At scale=1.0:
- `HealthyOtEnvironment`: alert records (5), ASM asset records (10), CVE records (5), IOC records (5)
- `CompromisedEndpoint`: alert records (20, ≥3 high-severity), ASM assets (10), CVEs (10), IOCs (10)
- `LargeScale`: alert (500), ASM asset (2000), CVE (1000), IOC (1000); total ~4500 records
- `DormantTenant`: 0 records across all surfaces
- Other archetypes: proportional to baseline; counts determined by `floor(baseline × scale)`

(traces to BC-3.4.002 postcondition — Cyberint schema source table)

### AC-002 — Each record validates against the correct sub-spec (traces to BC-3.4.002 postcondition — EC-007)

In `#[cfg(test)]`, alert records validate against `alert_api_specs.json`, ASM asset records
against `asm_assets_api_specs.json`, CVE records against `cve_api_specs.json`, IOC records
against `ioc_api_specs.json`. A record validated against the wrong sub-spec constitutes a
generator bug; test panics with surface type, spec path, and schema error.

### AC-003 — SchemaDrift targets one surface; schema_valid=false (traces to BC-3.4.002 postcondition for SchemaDrift)

For `SchemaDrift` archetype, exactly 1 record (at index 0 of the alert surface) intentionally
violates `alert_api_specs.json`. `provenance.schema_valid = false`. All records on other
surfaces remain valid.

### AC-004 — All record IDs carry org-slug prefix per surface type (traces to BC-3.4.004 postcondition — Cyberint row)

Alert records: `alert_id` starts with `"alert-{org_slug}-{seed}-"`.
ASM asset records: `asset_id` starts with `"dev-{org_slug}-{seed}-"`.
CVE and IOC records: primary ID field starts with `"alert-{org_slug}-{seed}-"` (alert-class IDs).

### AC-005 — Two calls with identical inputs are byte-identical (traces to BC-3.4.001 postcondition 1)

`generate(orgA, Cyberint, HealthyOtEnvironment, GenOpts::default())` called twice returns
byte-identical `records`. No `thread_rng` or timestamp entropy in call stack (BC-3.4.001 invariant 1).

### AC-006 — OrgA and OrgB ID sets are disjoint across all surfaces (traces to BC-3.4.004 postcondition 3)

ID intersection across all surfaces is empty for distinct org slugs. Verified by proptest.

### AC-007 — Schema validation gate absent from release build (traces to BC-3.4.002 invariant 4)

`cargo build --release` excludes all schema validation code. CI grep check confirms.

## Tasks

1. Read all 4 spec files in `.references/poller-express/docs/specs/` — identify primary ID
   field names and required fields per surface.
2. Implement `generate(org_id, archetype, opts) -> FixtureSet` in
   `crates/prism-dtu-cyberint/src/generator.rs` behind `#[cfg(feature = "fixture-gen")]`.
3. Model the 4 API surfaces as internal sub-generators sharing the same `seeded_rng` stream
   (advance RNG sequentially to ensure determinism across surfaces within one call).
4. Implement all 8 archetype branches; apply scale formula `floor(baseline × scale)` per surface.
5. Apply org-tagged IDs per BC-3.4.004 surface-specific field names.
6. `SchemaDrift`: mutate `alert_surface.records[0]` to violate `alert_api_specs.json`.
7. In `#[cfg(test)]`, validate each record against its correct sub-spec; panic on violation
   for non-SchemaDrift archetypes.
8. Write parameterised tests over all 8 archetypes × 4 surfaces.

## Architecture Compliance Rules

- Generator impl lives in `crates/prism-dtu-cyberint/src/generator.rs` only.
- Schema sources are the 4 files in `.references/poller-express/docs/specs/` — read-only.
- `#[cfg(feature = "fixture-gen")]` gate mandatory.
- Forbidden dependency: `prism-dtu-cyberint` must NOT depend on `prism-dtu-claroty`, `prism-dtu-armis`, or `prism-dtu-crowdstrike`.
- Single seeded RNG stream advances across all 4 surfaces — do NOT create 4 independent RNGs from the same seed.
