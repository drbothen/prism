# S-3.7.01 — Archetype catalog + GenOpts API

## Narrative

As a generator implementer, I want a canonical `Archetype` enum, `GenOpts` struct,
and `default_page_size()` per-sensor function in `prism-dtu-common` behind the
`fixture-gen` feature gate, so that all four sensor generator stories (S-3.7.02
through S-3.7.05) can share a single source of truth for archetype semantics,
RNG initialisation, and pagination baselines.

## Acceptance Criteria

### AC-001 — `Archetype` enum defines exactly 8 variants (traces to BC-3.4.003 precondition 1)

`pub enum Archetype` is `#[non_exhaustive]` and declares exactly:
`HealthyOtEnvironment`, `CompromisedEndpoint`, `AuthOutage`, `LargeScale`,
`PaginationEdgeCases`, `SchemaDrift`, `HighChurn`, `DormantTenant`.
A companion `all_archetypes() -> &'static [Archetype]` function provides
exhaustive iteration for parameterised tests (BC-3.4.003 invariant 2).

### AC-002 — `GenOpts` struct matches ADR-009 §2.3 signature (traces to BC-3.4.001 precondition 2–5)

`pub struct GenOpts { pub seed: u64, pub scale: f64, pub time_anchor: DateTime<Utc>, pub overrides: serde_json::Value }`
with `GenOpts::default()` returning `{ seed: 42, scale: 1.0, time_anchor: DateTime::UNIX_EPOCH, overrides: Value::Null }`.
`scale` is validated: must be positive and finite; `GenOpts::new()` returns
`Err(GenOptsError::InvalidScale)` if not satisfied (BC-3.4.001 precondition 3).

### AC-003 — RNG initialisation formula is exact (traces to BC-3.4.001 invariant 2)

`seeded_rng(seed: u64, org_id: &OrgId) -> ChaCha20Rng` computes
`org_id_hash = u64::from_le_bytes(org_id.as_bytes()[0..8])` and calls
`ChaCha20Rng::seed_from_u64(seed ^ org_id_hash)`.
No call to `rand::thread_rng()`, `SystemTime::now()`, or any non-deterministic
source exists anywhere in this module (BC-3.4.001 invariant 1).

### AC-004 — Override patch is JSON Merge Patch (RFC 7396) (traces to BC-3.4.001 postcondition 7)

`apply_overrides(base: serde_json::Value, patch: &serde_json::Value) -> serde_json::Value`
implements RFC 7396 merge semantics. A `null` patch value in the patch removes the key
from the base; a non-object patch replaces the entire base. Identical inputs always
produce identical output (determinism preserved through patching).

### AC-005 — `default_page_size()` per-sensor function provided (traces to BC-3.4.003 postcondition for PaginationEdgeCases)

`fn default_page_size(sensor: SensorType) -> usize` returns documented page-size constants:
values sourced from S-3.7.00 DERIVATION.md for Armis and CrowdStrike; from poller-bear and
poller-express SDK constants for Claroty and Cyberint. The function is used by
`PaginationEdgeCases` archetype to compute `page_size × 3` baseline count.

### AC-006 — `FixtureSet` and `Provenance` types defined (traces to BC-3.4.001 postcondition 1)

`pub struct FixtureSet { pub records: Vec<serde_json::Value>, pub cursors: Vec<String>, pub provenance: Provenance }`
`pub struct Provenance { pub org_id: OrgId, pub sensor_type: SensorType, pub archetype: Archetype, pub seed: u64, pub schema_valid: bool }`
`schema_valid` defaults to `true`; only `SchemaDrift` archetype sets it to `false`
(BC-3.4.002 postcondition for SchemaDrift).

### AC-007 — Feature gate: module absent from release builds (traces to BC-3.4.001 invariant 4)

All types in `crates/prism-dtu-common/src/generator/` are gated behind
`#[cfg(feature = "fixture-gen")]`. A `cargo build --release` of the workspace
without `--features fixture-gen` must not include any symbol from this module.
CI check: `grep -r "fixture_gen\|fixture-gen" target/release/` returns no hits.

## Architecture Compliance Rules

- Generator module lives in `crates/prism-dtu-common/src/generator/` — NOT a separate crate (D-056).
- Feature gate is `fixture-gen` — NOT `dtu` or `test`. Only activated in test/fixture builds.
- No production binary (`prism-mcp`, `prism-sensors`, etc.) may enable `fixture-gen` in its
  `Cargo.toml` feature list.
- `seeded_rng()` formula is locked: `seed ^ org_id_hash` — no deviation without ADR-009 amendment.

## Edge Cases

| ID | Description | Expected Behavior |
|----|-------------|-------------------|
| EC-001 | `scale = 0.0` passed to `GenOpts::new()` | Returns `Err(GenOptsError::InvalidScale)` — zero is not positive |
| EC-002 | `scale = f64::INFINITY` | Returns `Err(GenOptsError::InvalidScale)` — not finite |
| EC-003 | `seed = u64::MAX`, `org_id` with all-zero first 8 bytes | `seed ^ 0 = seed`; valid; produces deterministic stream |
| EC-004 | `apply_overrides` with `null` patch value for a key | Key removed from base (RFC 7396) |
| EC-005 | `all_archetypes()` count | Returns exactly 8 elements; verified in unit test |

## Test Plan
